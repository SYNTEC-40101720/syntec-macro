# 第一部分　基础概念

## 1. MACRO 程序概述

### 1.1 什么是 MACRO 程序

MACRO 程序是支持完整变量、运算、逻辑判断的加工程序格式。与标准 ISO 格式相比，MACRO 格式可以使用变量（`@1`、`#1`）、数学函数、条件判断（IF/CASE）、循环（FOR/WHILE/REPEAT）等高级语法，大幅增强控制器的应用弹性。

档案第一行以 `%` 开头，并加入关键字 `@MACRO`，控制器才会将该档案视为 MACRO 格式处理：

```iecst
%@MACRO
#1 := 1;
IF #1 = 1 THEN
  GOTO 10;
END_IF;
N10;
G01 G90 X50. Y0. F1000;
M30;
```

### 1.2 MACRO 与 ISO 格式的区别

| 特性 | ISO 格式 | MACRO 格式 |
|---|---|---|
| 变量支持 | 不支持（部分宏语法可能被误读） | 支持完整变量功能 |
| 行尾分号 | 可省略 | **必须加分号 `;`**（特定语法除外） |
| 注解括号 | `( ... )` | `(* ... *)` |
| 程序格式 | 仅支持车床习惯（C-type） | 无限制 |

**分号规则：**

- 除特定语法外，MACRO 程序每行**必须以分号 `;` 结尾**
- 10.120.45 之前：未加分号时会与下一行合并处理，若合并后合法则正常执行
- 10.120.45 之后：未加分号报 **COM-008**【子句中没有结束的符号' ; '】
- 10.120.61 之后：未加分号同样会与下一行合并（与 10.120.45 前一致）

**注解语法：**

```iecst
// 单行注解
(* 区块注解
   多行内容均不影响程序执行 *)
#1 := 100; // 注解可以写在语句后面
```

### 1.3 程序档案格式

程序内容第一行需使用 `%` 宣告为标题行，并加入关键字 `@MACRO`，才会被视为 MACRO 格式档处理。否则该档案将被视为 ISO 格式档处理。

#### ISO 格式档与 MACRO 格式档的差异

| 特性 | ISO 格式档 | MACRO 格式档 |
|---|---|---|
| MACRO 语法 | 不支援（部分宏语法可能被误读） | 支援完整变量功能 |
| 行尾分号 | 可省略 | **必须加分号 `;`**（特定语法除外） |
| 注解括号 | `( ... )` | `(* ... *)` |
| 程序格式 | 仅支持车床习惯（C-type） | 无限制 |

#### 分号规则

除特定语法外，MACRO 程序每行**必须以分号 `;` 结尾**：

- 10.120.45 之前：未加分号时会与下一行合并处理，若合并后合法则正常执行
- **10.120.45 之后**：未加分号报 **COM-008**【子句中没有结束的符号' ; '】
- 10.120.61 之后：未加分号同样会与下一行合并（与 10.120.45 前一致）

#### 60KB 档案限制（多轴群副程序）

被当作副程序（子程序）或宏程序呼叫的加工档，建议不要使用多轴群档案（含轴群辨识符号 `$1`、`$2`）。多轴群副程序档案必须小于 **60KB (60000 bytes)**，否则报 **COR-203**【加工程序格式不符】。单轴群副程序则无此限制。

#### 版本行为（10.120.32）

- **10.120.32 之前**：不支援 IF、CASE、REPEAT、FOR、WHILE 等有范围区间的语法，否则报 **COR-204**【档案太大】
- **10.120.32 及之后**：支援上述语法
- 执行加工前建议执行图形模拟进行语法检查
- 语法范围区间过大时，加工时有可能会使轴向降速。可查看诊断变数 D320「解译总线单节数」及 D376「运动规划减速次数」判断是否为语法区间过大造成降速

#### 字元编码

除了程序注解、使用字串做为引数的 MACRO 语法以外，**仅允许以 ASCII 撰写加工程序**。若使用了 ASCII 以外的字符，将报 **COM-027**【无效的字元】。

支援 CR (`"\r"`)、LF (`"\n"`)、CR+LF (`"\r\n"`) 三种换行方式。

#### 代码区块范例

```iecst
%@MACRO
#1 = 1;
IF #1 = 2 THEN
  // =====================================
  // IF 条件不成立，此处程序不执行
  // =====================================
END_IF;
G01 X100. Y100.;
```

### 1.4 轴群辨识符号

`$1`、`$2`、`$3`、`$4` 用于在同一份加工程序中区分不同轴群的执行内容：

```iecst
%
$1 轴群一加工程序...
$2 轴群二加工程序....
%
```

注意：`$1`、`$2` 等**不可置于两个 `%` 符号之外**，否则报警 COR-210。

### 1.5 结束标记一览

| 语法 | 结束标记 |
|---|---|
| IF | `END_IF;` 或 `ENDIF;` |
| CASE | `END_CASE;` 或 `ENDCASE;` |
| FOR | `END_FOR;` 或 `ENDFOR;` |
| WHILE | `END_WHILE;` 或 `ENDWHILE;` |
| REPEAT | `END_REPEAT;` 或 `ENDREPEAT;` |

### 1.6 嵌套层级限制

CASE / IF / REPEAT / WHILE / FOR 语法支持嵌套，**最高 10 层**。超过 10 层将报 **COM-007**【重复循环太深】。

### ATAN2
计算 Y/X 的反正切值，并根据输入引数 (X,Y) 所处的象限位置决定输出结果。计算角度范围介于 ±180°。

**注意事项：**
- 引数 X 和引数 Y 必须为数字，否则跳警报 COR-023

## 变数/资料存取函数

### GETARG
读取呼叫者传递的引数。

**语法：** `GETARG(引数名称)`

**范例：**
```macros
// 假设 00001 主程序内容为：G101 X30. Y40. Z1=40. Z2=50;
// G0101 扩充宏程序使用 GETARG 读取引数内容
#1 := GETARG(X);     // 将 X 引数内容 30. 存到 #1
#2 := GETARG(Z1);    // 将 Z1 引数内容 40. 存到 #2
#3 := GETARG(W);     // 因 W 不存在，所以 #3 内容为 VACANT (#0)
```

---

### GETTRAPARG
读取 G66/G66.1 在 Trap 单节内的引数内容。

**语法：** `GETTRAPARG(引数名称)`

**范例：**
```macros
// 假设 00001 主程序内容为：
// G66 P100 X100. Y100.
// G01 X20.
// 00100 副程序使用 GETTRAPARG 读取引数内容
#1 := GETARG(X);          // 将 X 引数内容 100. 存到 #1
#2 := GETTRAPARG(X);      // 将 Trap 单节 X 引数内容 20. 存到 #2
```

---

### SCANTEXT
读取公用变数所储存的字串内容。将字串存入公用变数时，控制器会先进行 ASCII 转码，因此需使用此函数以取得正确字串内容。

**语法：** `SCANTEXT(变数号码)`

**范例：**
```macros
%@MACRO
@1 := "12";     // 16进位 HEX=3231, 10进位 DEC=12849
#1 := SCANTEXT(1);
OPEN("NC");
PRINT("@1");
PRINT("#1");
CLOSE();
M30;
// 所得结果为：@1 = 12849, #1 = 12
```

---

### STR2INT
转换数字字串为整数型态。

**语法：** `STR2INT("字串")`

**范例：**
```macros
// EX1:
@1 := "5555";
#1 := SCANTEXT(1);       // #1 = 字串5555
#2 := STR2INT("#1");     // #2 = 5555

// EX2:
#1 := STR2INT("100");    // #1 = 100

// EX3:

```

**注意事项：** 只要有文字的部分，即为不合法之输入。

### SYSVAR

- 轴群识别码：1 为第一轴群组，2 为第二轴群组，以此类推
- 系统变数码：欲读取的系统变数编号

**范例：**
```macros

```

---

### SYSDATA
读取系统诊断变数。

**语法：** `SYSDATA(系统诊断变数号码)`

**有效版本：** 10.118.23U, 10.118.28H, 10.118.33

**范例：**
```macros
WAIT();                    // 挡预解，否则无法取得当前最新的数值
#1 := SYSDATA(336);        // 轴卡交换时间 (D336)
#2 := SYSDATA(77);         // 硬件剩余存储器 (D77)
OPEN("DbgData.txt", "a");  // 开档，档名为 "DbgData.txt"
PRINT("#1#2");             // 将变数值输出到档案内
CLOSE();                   // 关档
```

**注意事项：**
- 执行函数前建议下 WAIT() 函数挡预解
- 引数格式须为整数
- 变数号码须在诊断变数号码范围内，否则发 COR-016

### PARAM
读取系统参数的内容，可透过单引数读取 Pr 参数，或透过双引数读取 V12 PN 参数。

**语法：** `PARAM(参数号码)` 或 `PARAM(参数号码, 轴群)`

**范例：**
```macros
#1 := PARAM(3204);              // 读取 Pr3204 (PLC扫描时间) 之内容
#2 := PARAM(20001, 2);          // 读取 P20001 (机床型态)，N=2 (第二轴群)
```

**注意事项：**
- 双引数有效版本：12.0.6 及之后的版本
- 若读取的参数不支援则回传 Vacant
- 单引数回传型态仅支援整数，双引数回传型态支援整数、浮点数、字串

### STDAX
将整数数值转成对应轴向的标准单位。

**语法：** `STDAX(变数, 轴向名称)`

**范例：**
```macros
#24 := STDAX(#24, X);
#3 := STDAX(#3, A);
```

**注意事项：**
- 若输入数值为小数点型态，则不进行任何数值或型别转换
- 若输入数值为整数型态，且 Pr3241=1，则不进行任何数值转换，仅将输入数值型别转换为小数点型态

### AXID

**范例：**


## 标准单位转换函数

### STD
根据 Pr17 将整数数值转换成当时系统设定的输入单位 (Input Unit, IU)。

**语法：** `STD(引数1, 引数2)`

- 引数1：欲改变单位之数值
- 引数2：标准化单位，一般使用 #1600（#1600 设定值来自 Pr17）

**范例（公制）：**
```macros
// 当 Pr17=2，#1600 对应 LIU = 0.001mm
#9 := 100;
#10 := STD(#9, #1600);   // #9 为 100 个 BLU，故 #10 为 0.1mm (100*0.001)

// 当 Pr17=3，#1600 对应 LIU = 0.0001mm
#9 := 100;
#10 := STD(#9, #1600);   // #9 为 100 个 BLU，故 #10 为 0.01mm (100*0.0001)
```

**范例（英制）：**
```macros
// 当 Pr17=2，#1600 对应 LIU = 0.0001 inch
#9 := 100;
#10 := STD(#9, #1600);   // #9 为 100 个 BLU，故 #10 为 0.01inch (100*0.0001)
```

**注意事项：**
- 若输入数值为小数点型态，则不进行任何数值或型别转换
- 若输入数值为整数型态，且 Pr3241=1，则不进行任何数值转换，仅将输入数值型别转换为小数点型态

## 堆栈操作函数

### PUSH
将资料塞进栈 (STACK)，最先 PUSH 进入控制器的数值会栈在最底层，并依序往上叠加。

**语法：** `PUSH(数值)`

**范例：**
```macros
PUSH(#1);   // 将变数 #1 塞入栈中
```

---

### POP
将栈 (STACK) 里面的资料取出，依序由最上层一路取到最底层。

**语法：** `POP()`

**范例：**
```macros
PUSH(5);   // 将数字5塞入栈中
#1 := POP();   // 取出栈中最上层数值 (#1=5)
```

---

### STKTOP
将栈 (STACK) 里面的资料复制出来（不删除）。

**语法：** `STKTOP[索引]`

**范例：**
```macros
PUSH(5);   // 将数字5塞入栈中
PUSH(6);   // 将数字6塞入栈中
PUSH(7);   // 将数字7塞入栈中
#1 := STKTOP[0];   // #1 = 7
#2 := STKTOP[1];   // #2 = 6
#3 := STKTOP[2];   // #3 = 5
```

## I/O 控制函数

### READDI
读取指定的 DI 点编号。

**有效版本：** 10.116.23

```macros
@52 := READDI(31);             // 把 I31 的值读出来填到 @52
```

### SETDO
设定指定的 DO 点开或关。

**语法：** `SETDO(点编号, 开或关)`

- 开或关：1 = 开，0 = 关

**有效版本：** 10.116.23

**范例：**
```macros
SETDO(3, 1);   // 设定 D03 on
SETDO(8, 0);   // 设定 D08 off
```

## 档案操作函数

### OPEN
在 NcFiles 或 Macro 资料夹开启一文字档。

**语法：** `OPEN("档名")` 或 `OPEN("档名", "写档方式")`

**开档规则：**
- 档案已存在：开启原本的档案
- 档案不存在：生成一份新的在 NcFiles 资料夹
- 若档案名称为 "COM" 时，表示打开 RS232/RS485 传输埠

**写档方式：**
- `"a"`：保留原先的档案内容，将新资料接续输出
- `"w"` 或不指定：清空原先的内容，重新输出

**范例：**
```macros

END_FOR;
PRINT("\p");   // 输出 % 字符
CLOSE();       // 关闭传输埠

// 保留原先档案内容
OPEN("PROBE.NC", "a");

// 清空原先档案内容
OPEN("PROBE.NC");
OPEN("PROBE.NC", "w");
```

**注意事项：**
- 写档方式指定错误会发出 COR-301 警报
- 使用 # 或 @ 变数转成字串，小数点输出位数会与 Pr17 连动（10.118.12C 起）

## 流程控制函数

### SLEEP
暂时放弃此次宏程序循环的执行权，一般配合回圈使用，以避免进入无穷回圈而造成人机卡死。

**主要功用：**
- **防止系统卡死**：在 MACRO 程式中使用迴圈（如 WHILE、FOR、REPEAT）时，若进入无穷迴圈，高优先权的核心执行绪会佔据 CPU 资源，导致人机介面锁死或无法操作。
- **释放执行权**：执行 SLEEP() 会通知系统核心暂时放弃本次解译权，让出资源给其他执行绪（如人机介面），并在下一次解译週期（约数十毫秒后）继续执行。
- **配合 PLC 交握**：常用于等待 PLC 完成特定任务（例如等待某个 R 值改变），在等待期间使用 SLEEP() 可达成等待效果且不佔用系统效能。

**使用建议与限制：**
- **建议场合**：强烈建议在所有迴圈结构中加入 SLEEP()，以确保在异常发生时仍能透过人机介面停止程式。
- **无效场合**：SLEEP() 仅针对迴圈有效，若放在 IF 判断式中则无实质作用（因为 IF 仅执行单次判断）。
- **潜在风险**：在特定版本或情境下（如机器人平滑指令），使用 SLEEP() 偶发可能导致系统 Crash 或报警（如 OP-1）。

**与 WAIT 的区别：**
- **WAIT**：主要用于关闭「预解」功能，确保 MACRO 读取到最新的系统状态（如 R 值），通常加在变数存取之后。
- **SLEEP**：主要用于「休息」与资源分配，防止迴圈耗尽 CPU 资源。

**语法：** `SLEEP()`

**范例：**
```macros
WHILE #1 = 0 DO
  SLEEP();  // 释放执行权，防止系统卡死
END_WHILE;
```

**建议：** 在 REPEAT/WHILE/FOR/GOTO 等回圈中适时加入 SLEEP() 函数。

---

### WAIT
系统停止预解，直到 WAIT 之前的指令执行完毕。

**语法：** `WAIT()`

**范例：**
```macros
%@MACRO
@50 := 1;                      // @50 内容填成 1
G90 G01 X100. F1000;           // 假设此时 Reset 系统
WAIT();
@50 := 0;                      // @50 内容填成 0
M30;
// 假设在 G01 尚未执行完毕时复位系统，由于下一个单节为 WAIT，
// 因此系统停止预解，故复位后 @50 内容仍然是 1。
```

**注意事项：** WAIT 只确保 WAIT 前的 G/M 码执行完毕前不会继续往后预解（其中 M98、M99、M198 为例外），对其他指令则不会有作用。

### 2.1 变量系统

MACRO 程序中的变量分为三类：**区域变量**、**公用变量**、**系统变量**，另有 **轴群记忆变量**和 **App 专用变量**。

### 2.1.1 区域变量 #

区域变量编号 `#0` ~ `#400`，生命周期仅限当前 MACRO 执行期间，执行完毕后自动恢复为 VACANT（空白）。

| 编号 | 说明 | 类型 |
|---|---|---|
| `#0` | 空白（VACANT） | 只读 |
| `#1` ~ `#26` | 系统内定作为宏程序引数使用 | 读写 |
| `#27` ~ `#400` | 宏程序与副程序共用 | 读写 |

副程序可与父程序共用区域变量，生命周期随父程序结束。

**直接赋值：**

```iecst
@1 := 123;   //冒号+等号
#1 := 456;
```

**间接赋值（中括号语法）：**

```iecst
#1 := 123;
@[#1] := 567;    // @123 = 567
@[#1+7] := 890; // @130 = 890
```

### 2.2 公用变量 @

公用变量编号 `@0` ~ `@165535`，生命周期仅受关机影响。

| 编号范围 | 说明 | 类型 |
|---|---|---|
| `@0` | 空白（VACANT） | 只读 |
| `@1` ~ `@400` | 一般运算用变量 | Double |
| `@401` ~ `@655` | 对应寄存器 R1~R255 | Long |
| `@656` ~ `@999` | 有记忆性变量（关机后保留） | Double |
| `@1000` | 系统变量（10.118.56Z 后改为有记忆性） | Double |
| `@1001` ~ `@1999` | 有记忆性变量（关机后保留） | Double |
| `@10000` ~ `@14095` | 对应寄存器 R0~R4095 | Long |
| `@60000` ~ `@79999` | 扩充公用变量（仅 CE 系统适用） | Double |
| `@100000` ~ `@165535` | 对应寄存器 R0~R65535 | Long |

### 2.3 轴群使用者记忆变量 #401 ~ #999

| 轴群 | 变量占用范围 |
|---|---|
| 第一轴群 | L42000 ~ L43197 |
| 第二轴群 | L47000 ~ L48197 |
| 第三轴群 | L52000 ~ L53197 |
| 第四轴群 | L57000 ~ L58197 |

- 预设值为 0；支援版本：10.120.42.x、10.120.52.x、10.120.x、12.0.x 及之后版本
- 降版至 10.118.86M 等之前版本后升级，数值会被清 0

### 2.4 App 专用变量 AR / MAR

AR 为 App 区域变量（区域有效），MAR 为 App 永久储存变量（关机后仍保留）：

```iecst
AR1 := 789;   // App 区域变量
MAR1 := 789;  // App 永久储存变量（关机后仍保留）
#1 := 123;
AR[#1] := 789;  // 间接定值：AR123 = 789
MAR[#1] := 789; // 间接定值：MAR123 = 789
```

**支援版本：** 10.118.39 及之后版本

**限制：**
1. 仅 **APP 专用 MACRO** 才可存取 AR/MAR；在其他程序呼叫 → 跳 **COR-016**
2. 变数号码为 **0-Based**（即 AR0、AR1、AR2……）
3. 若存取超出范围（如 MAR 个数只有 512，却下 `MAR512 := 1;`）→ 跳 **COR-016**
4. 直接存取负值位址（如 `MAR-1 := 1;`）→ 跳 **COM-003**
5. 直接存取小数位址（如 `MAR1.1 := 1;`）→ 跳 **COM-003**
6. 间接存取负值位址（如 `MAR[-1] := 1;`）→ 跳 **COR-016**
7. 间接存取小数位址（如 `MAR[1.1] := 1;`）→ 跳 **COR-016**
8. 均**不支援存字串**；仅支援 MACRO 格式，不支援 ISO 格式

> ⚠️ **待确认：** AR 和 MAR 的实际最大个数（目前已知 MAR 至少 512 个，更高编号是否视版本而定？）

## 3. 流程控制语法

### 3.1 IF 条件判断

```iecst
IF <条件> THEN
  <语句>
ELSEIF <条件> THEN
  <语句>
ELSE
  <语句>
END_IF;
```

- 结束标记：`END_IF;` 或 `ENDIF;`
- 限制：每档最多 256 组（10.120.32+）/ 512 组（10.120.44A+）

### 3.2 CASE 多条件判断

```iecst
CASE <条件变量> OF
<值>: <语句>
<值1>,<值2>: <语句>
ELSE <语句>
END_CASE;
```

- 条件变量需为**大于等于 0 的整数**
- 结束标记：`END_CASE;` 或 `ENDCASE;`

### 3.3 FOR 循环

```iecst
FOR <变量> := <起始> TO <终止> [BY <步进>] DO
  <语句>
END_FOR;
```

> ⚠️ 注意不要在 FOR 循环内使用会跳转进出的命令（如 GOTO 出循环再跳回、M98 等），否则累加次数会异常。

### 3.4 WHILE 循环

```iecst
WHILE <条件> DO
  <语句>
END_WHILE;
```

### 3.5 REPEAT 循环

```iecst
REPEAT
  <语句>
UNTIL <条件>
END_REPEAT;
```

- 先执行循环体，再判断条件（第 1 次先执行后判断）

### 3.6 EXIT — 跳出循环

```iecst
IF (#14 = 4) & (#16 = 1) THEN
  EXIT; // 直接跳离当前循环
END_IF;
```

### 3.7 GOTO 跳转

```iecst
GOTO n;
```

- 需配合单节次序码（N 码）使用，跳到指定 N 行号执行
- 若存在两个相同 N 行号，以第一个为准

> ⚠️ **警告：** 使用 GOTO / FOR / WHILE / REPEAT 时，应避免无穷循环。建议在循环中适时加入 `SLEEP()` 函数，以便在误入无穷循环时仍可通过人机画面中止程序。

# 第二部分　移动指令

## 5. MOVJ — 关节运动

控制器针对各轴独立进行差补，让手臂末端点到达目标位置。注意：运动过程中末端点的路径**并非直线**，而是各轴同时启动、同时到达的关节轨迹。

**对应 G 码：** G53.15（10.118.27 之前为 G1.2）
**支援版本：** 10.118.0A 及之后版本

### 指令格式

```
MOVJ [C1=_] [C2=_] [C3=_] [C4=_] [C5=_] [C6=_]
     [A1=_] [A2=_] [A3=_] [A4=_] [A5=_] [A6=_]
     [Q=_] [FJ=_] [FEJ=_] [PL=_] [ACC=_] [DEC=_] [SKIP];
```

| 引数 | 说明 | 单位 | 范围 |
|---|---|---|---|
| C1~C6 | 每个内部轴的目标角度或位置 | mm / deg | — |
| A1~A6 | 每个外部轴的目标角度或位置 | mm / deg | — |
| Q | 指定工具编号 | — | 0~20 |
| FJ | 内部轴单轴速度百分比 | % | 0.01~100 |
| FEJ | 外部轴单轴速度百分比 | % | 0.01~100 |
| PL | 轨迹平滑等级（详见第三部分） | — | 0~10 |
| ACC/DEC | 加减速度倍率 | % | 20~100 |
| SKIP | 指定该单节为可跳脱 | — | — |

### 替代语法（教导包装）

```
G53.102 [C1=_] [C2=_] ...
```

支援版本：10.118.40G、10.118.44 及之后版本

---

## 6. MOVJ-II — 关节运动（末端位置输入）

与 MOVJ 的区别在于：目标位置由**末端点在空间中的坐标（X/Y/Z/A/B/C）**指定，控制器自行逆解算各轴角度。

**对应 G 码：** G53.16（10.118.27 之前为 G1.3）
**支援版本：** 10.118.0A 及之后版本

### 指令格式

```
MOVJ-II [LP=_] [GP=_] X=_ Y=_ Z=_ A=_ B=_ C=_
        [A1=_] [A2=_] [A3=_] [A4=_] [A5=_] [A6=_]
        [I=_] [W1=_] [W2=_] [W3=_] [W4=_] [W5=_] [W6=_]
        [P=_] [Q=_] [FJ=_] [FEJ=_] [PL=_] [ACC=_] [DEC=_] [SKIP];
```

| 额外引数 | 说明 |
|---|---|
| LP / GP | 区域/全域参考点编号（指定目标位置） |
| X/Y/Z | 末端点在空间中的位置（mm） |
| A/B/C | 末端点旋转姿态（deg） |
| I | 姿势解域指定值（10.118.86+，根据构型） |
| W1~W6 | 旋转关节同界角指定值，范围 -1~1 |

## 7. MOVL — 末端直线运动

末端点以**线性轨迹**从当前位置移动到目标位置，速度恒定。

**对应 G 码：** G1.15（10.118.27 之前为 G1.1）

### 指令格式

```
     [P=_] [Q=_] [FL=_] [FR=_] [FEJ=_]
     [PL=_] [PQ=_] [PR=_] [ACC=_] [DEC=_] [SKIP];
```

| 额外引数 | 说明 | 单位 | 范围 |
|---|---|---|---|
| LP/GP | 区域/全域参考点编号 | — | LP:1~1000 / GP:1~100 |
| FL | 末端点线性方向进给速度 | mm/s | 0.1~Pr405 |

## 8. MOVC — 圆弧运动

末端点在空间中以**圆弧轨迹**移动。由上一单节终点（起点）、圆弧通过点和结束点，三点定义完整空间圆弧。

**对应 G 码：** G2.15（10.118.27 之前为 G1.4）
**支援版本：** 10.118.0A 及之后版本（完整格式 10.118.52H、10.118.56B、10.118.57+）

### 指令格式

```
MOVC [LP=_] [GP=_] Xp=_ Yp=_ Zp=_ Ap=_ Bp=_ Cp=_
                   X=_ Y=_ Z=_ A=_ B=_ C=_
     [A1=_] [A2=_] [A3=_] [A4=_] [A5=_] [A6=_]
     [P=_] [Q=_] [FL=_] [FR=_] [FEJ=_]
     [PL=_] [PQ=_] [PR=_] [ACC=_] [DEC=_];
```

- 带 `p` 后缀的坐标为**圆弧通过点**
- 无 `p` 后缀的坐标为**圆弧结束点**
- 起点由上一单节终点决定
- 圆弧模式 P0 & P1 下，协同功能关闭时外部轴不参考中间点

# 第三部分　速度与轨迹

## 13. FJ / FEJ / FL / FR — 速度参数

指定机械手臂的移动速度。

**支援版本：** 10.118.0A 及之后版本

| 引数 | 说明 | 单位 | 范围 |
|---|---|---|---|
| FJ | 内部轴（机械臂轴向）单轴速度百分比 | % | 0.01~100 |
| FEJ | 外部轴（非机械臂轴向）单轴速度百分比 | % | 0.01~100 |
| FL | 末端点线性方向进给速度 | mm/s | 0.1~Pr405 |
| FR | 末端点旋转方向进给速度 | deg/s | 0.1~Pr4307 |

**使用说明：**
- 单节可混合输入 FJ、FEJ、FL、FR，也可任选其一或组合
- 超过 100% 自动视为 100%，小于 0.01 自动视为 0.01
- FL 与 FR 由于需维持同动性，运行中不一定总能达到设定速限

**对应指令组合：**
- FJ / FEJ：MOVJ 系列使用
- FL / FR：MOVL、MOVC 使用

---

## 14. PL / PQ / PR — 转角平滑

指定机械手臂在转角处的平滑程度，减少冲击和振动。

**支援版本：** 10.118.0A 及之后版本

| 引数 | 说明 | 单位 | 范围 |
|---|---|---|---|
| PL | 轨迹平滑等级 | — | 0~10 |
| PQ | 平滑轨迹开始时离目标点的**位置**距离 | mm | 0~半个单节长度 |
| PR | 平滑轨迹开始时离目标点的**姿态**角度 | deg | 0~半个单节旋转角度 |

### 14.1 PL 等级说明

| 等级 | 说明 |
|---|---|
| 0 | 准确到达点位，速度减速至零 |
| 1~9 | 逐级增加平滑程度，每格代表 10% |
| 10 | 最大平滑（100% 加减速时间就开始 Overlapping） |

- PL 不受小数点影响，`PL10` 与 `PL10.` 相同
- PL 超出范围时自动取边界值（负值视为 0，超过 10 视为 10）

### 14.2 PQ / PR 说明

- PQ：末端点位置距离目标点多少 mm 时开始与下一单节平滑
- PR：末端点姿态距离目标姿态多少 deg 时开始平滑
- 若单节在加减速时间内的位移量小于 PQ/PR，则取该实际位移量（物理限制）

### 14.3 作用规则

- 程序预设值为零，无平滑指令时每个目标点都会减速至零
- 平滑指令在程序内采**后令盖前令**规则
- PQ 与 PR 仅对 MOVL、MOVC 有效；对 MOVJ 无意义，系统自动设为 PL0
- PL 对 MOVJ、MOVL、MOVC 均有效

### 14.4 注意事项

- 单行 MOVL 指令附带平滑指令时，PL、PQ、PR **只能选择一个**，否则报 **RBT-103** 警报
- PQ 与 PR 为对末端轨迹位置与姿态的控制

---

## 15. ACC / DEC — 加减速度

指定机械手臂的加减速度倍率。

**支援版本：** 10.118.25 及之后版本

```
ACC_ DEC_;
```

| 引数 | 说明 | 单位 | 范围 |
|---|---|---|---|
| ACC | 加速度倍率 | % | 20~100 |
| DEC | 减速度倍率 | % | 20~100 |

### 15.1 使用规则

- 单节可混合输入 ACC、DEC，也可只取一个，顺序不限
- 采**后令盖前令**规则，连续下多行时采用最后一行
- 按下 **RESET** 时，ACC、DEC 倍率初始化为 100%
- 加减速度倍率是相对于压速的加速度（机器人轴、外部轴、末端直线、末端旋转皆有可能压速）
- ACC 与 DEC 设定相同时，代表加减速度相同（参考同一个原始加速度）

### 15.2 与移动指令的组合

ACC、DEC 可单独下指令，也可搭配运动指令使用：

```iecst
// 单独下达
ACC_;
DEC_;

// 搭配运动指令
MOVJ C1=_ C2=_ ... ACC_ DEC_;
MOVL X=_ Y=_ ... ACC_ DEC_;
INCMOVJ C1=_ C2=_ ... ACC_ DEC_;
INCMOVL X=_ Y=_ ... ACC_ DEC_;
```

### 15.3 注意事项

- 输入超出范围时报 **RBT-102** 警报
- ACC、DEC 倍率是根据运动参数对应的加减速度计算而来，修改运动参数会影响实际加减速度
- 例：加减速时间 100ms + ACC 50% 等效于 加减速时间 200ms + ACC 100%

## 16. USERCOR — 呼叫用户坐标系

## 20. SKIPCOND / SKIP — 跳脱功能

在指定的运动单节中，当满足条件时立即停止当前运动，跳到下一单节执行。用于焊接寻位、触碰感测等应用。

**支援版本：** 10.118.33、10.118.28G 及之后版本

### 20.1 指令格式

```
SKIPCOND E_ Q_ R_ P_;   // 设定跳脱条件
SKIP;                    // 指定该运动单节可进行跳脱
```

| 引数 | 说明 | 范围 |
|---|---|---|
| E | 讯号源：1=I-bit / 2=C-bit / 3=R-bit | 1~3 |
| Q | 讯号号码 | 见下 |
| R | 讯号状态：0=OFF / 1=ON | 0~1 |
| P | 刹车模式：0=急煞 / 1=软煞 | 0~1 |

**Q 讯号号码格式：**
- E=1（I-bit）：Q 0~511
- E=2（C-bit）：Q 0~511
- E=3（R-bit）：Q 0~65535，格式为 `R码×100 + 位`
  - R53.12 → `E3 Q5312`
  - R65.00 → `E3 Q6500`

**刹车模式：**
- P=0（急煞）：控制器立即停止发送命令，关节以最快速度停止
- P=1（软煞）：按参数设定的加减速时间和 Jerk 沿原路径轨迹煞停

### 20.2 典型用法

```iecst
SKIPCOND E1 Q33 R1 P1;   // 设定 I33 为 ON 时触发软刹车跳脱
MOVL X100. Y0. Z0. SKIP; // 指定此单节可跳脱
N10;                       // 跳脱后跳到这里继续执行
```

> **注意：** R 为状态检查（非边缘触发），因此若讯号在手臂到达前已为 ON，跳脱仍会触发。

## 25. WAIT / PAUSE — 程序控制

### 25.1 PAUSE — 暂停

暂停当前程序执行，需要人工重新启动。

```
PAUSE;
```

### 25.2 辅助函数 SLEEP

SLEEP() 是新代控制器 MACRO 程式中的重要指令，主要功用与特性如下：

**主要功用：**

- **防止系统卡死**：在 MACRO 程式中使用迴圈（如 WHILE、FOR、REPEAT）时，若进入无穷迴圈，高优先权的核心执行绪会佔据 CPU 资源，导致人机介面（萤幕）锁死或无法操作。
- **释放执行权**：执行 SLEEP() 会通知系统核心暂时放弃本次解译权，让出资源给其他执行绪（如人机介面），并在下一次解译週期（约数十毫秒后）继续执行。
- **配合 PLC 交握**：常用于等待 PLC 完成特定任务（例如等待某个 R 值改变），在等待期间使用 SLEEP() 可达成等待效果且不佔用系统效能。

**使用建议与限制：**

- **建议场合**：强烈建议在所有迴圈结构中加入 SLEEP()，以确保在异常发生时仍能透过人机介面停止程式。
- **无效场合**：SLEEP() 仅针对迴圈有效，若放在 IF 判断式中则无实质作用（因为 IF 仅执行单次判断）。
- **潜在风险**：在特定版本或情境下（如机器人平滑指令），使用 SLEEP() 偶发可能导致系统 Crash 或报警（如 OP-1）。

**与 WAIT 的区别：**

- **WAIT**：主要用于关闭「预解」功能，确保 MACRO 读取到最新的系统状态（如 R 值），通常加在变数存取之后。
- **SLEEP**：主要用于「休息」与资源分配，防止迴圈耗尽 CPU 资源。

**语法：** `SLEEP()`

**范例：**
```macros
WHILE #1 = 0 DO
  SLEEP();  // 释放执行权，防止系统卡死
END_WHILE;
```

## 29. G192.1 / G192.2 — 末端追踪

3. 追踪过程中不支援反向执行

## 30. 变量型态转换

### 30.1 Double 与 Long 的区别

| 类型 | 说明 | 运算 |
|---|---|---|
| Double | 64 位浮点数，支持小数 | 整数除法会截断小数部分 |
| Long | 32 位整数 | MOD 运算仅支持 Long 类型 |

### 30.2 整数除法的陷阱

整数除法规则（来源：Macro 手册）：
> 使用除号「/」时，若**分子与分母都是整数**，所得结果仍为整数。整数与非整数的区别在于有无加入小数点。

| 运算 | 结果 | 说明 |
|---|---|---|
| `1 / 2` | `0` | 整数 ÷ 整数 → 整数（截断小数） |
| `1 / 2.0` | `0.5` | 分母带小数点 → 浮点除法 |
| `1. / 2` | `0.5` | 分子带小数点 → 浮点除法 |
| `(1/2) * 1.0` | `0.0` | 刮号内均为整数，先算整数结果再乘 |

> ⚠️ **待确认：** 负数 ÷ 正整数（如 `-3 / 2`）的取整方向（趋近零还是趋近负无穷）。
> ⚠️ **待确认：** 除以零时的系统行为（跳警报？结果定义？）。
> ⚠️ **待确认：** Long 类型变量执行除法时的溢出处理。

### 30.3 寄存器对应变数型态

| 变量 | 对应寄存器 | 类型 |
|---|---|---|
| `@401` ~ `@655` | R1~R255 | Long |
| `@10000` ~ `@14095` | R0~R4095 | Long |
| `@100000` ~ `@165535` | R0~R65535 | Long |

> **注意：** `@1` ~ `@400` 和 `@656` ~ `@999`、`@1001` ~ `@1999` 均为 Double 类型。DOS 仅支援 R0~R7999，超出范围的寄存器对应变量在 DOS 环境下不可用。

## 38. MACRO 自订警报（ALARM）

MACRO 程序可主动触发自定义警报，提醒操作者注意特定状态。

### 语法

```iecst
%@MACRO
ALARM(xxx);  // xxx 为警报代码（整数）
ALARM(xxx, "讯息内容");  // 可附加字串说明
```

- 警报代码为整数；MACRO 警报发生时，必须**复位系统**才可消除警报
- 警报 ID 允许范围：0 ~ 65535；超出范围报 **COR-023**【语意错误】
- 10.120.62Z 之前版本：警报 ID 上限为 32767
- 10.120.62Z 及之后版本：警报 ID 上限为 65535；预设 ID 为 65535（之前版本为 -1）

### 与系统警报的差异

MACRO 警报与系统警报（CORSYN.DGD 中定义的警报）的差异如下：

| 项目 | MACRO 警报 | 系统警报 |
|---|---|---|
| 复位方式 | 需手动复位系统 | 同 |
| 档案依赖 | 无 | 需维护警报定义档 |
| 灵活性 | 可动态产生 | 需预先定义 |

## 39. MACRO 自订提示（MSG）

MACRO 程序可主动显示提示讯息，提醒操作者注意特定状态。

### 语法

```iecst
MSG(提示ID);
MSG("提示显示内容");
MSG(提示ID, "显示内容");
```

- 提示内容有字串长度限制：**中文 19 个字，英文 39 个字**
- 提示 ID 允许范围：0 ~ 65535；超出范围报 **COR-023**【语意错误】
- 提示讯息显示后，程序继续执行（不像 ALARM 会暂停）
- 若提示档案（MsgLib.txt）不存在或提示 ID 不存在，显示空白

### MSG 与 ALARM 的区别

| 特性 | MSG | ALARM |
|---|---|---|
| 是否需要复位 | 否 | 是 |
| 是否暂停程序 | 否 | 是 |
| 显示字数限制 | 中文19/英文39字 | 无（直接显示） |
| 适用场景 | 资讯提示、操作引导 | 错误提醒、紧急状况 |
