## 当前架构入口

### 扩展入口

- `src/extension.js`
  - VS Code provider 注册
  - diagnostics collection
  - CodeAction provider
  - Definition / References / Workspace Symbol / Hover / Completion / Formatting
- `src/navigationSymbols.js`
  - `%@MACRO` 与 N 标签提取
  - 静态 G/O 程序名识别
  - 静态宏调用解析与动态目标边界
  - 单文件导航索引与跨平台路径处理

### 诊断核心

- `src/validator.js`
  - 主验证流程
  - 行上下文创建
  - 行级诊断规则注册
- `src/diagnosticCodes.js`
  - 所有稳定诊断 code
- `src/diagnosticActions.js`
  - Quick Fix 替换规则
  - 说明型 action 文案
  - 控制流闭合词映射
- `src/diagnosticFactory.js`
  - 诊断对象工厂
  - 诊断去重 key
  - warning-over-error 抑制
  - 诊断排序
- `src/diagnosticRules.js`
  - 行级规则注册 helper
- `src/statementClassifier.js`
  - 语句分类
  - 行尾分号信息

### 分模块诊断

- `src/controlFlowValidator.js`
  - IF/FOR/WHILE/CASE/REPEAT 栈配对
  - CASE 分支风格提示
- `src/functionArgumentValidator.js`
  - 内置函数静态参数检查
- `src/robotValidator.js`
  - 机器人/LTP 单行语法诊断
  - 机器人/LTP 跨行状态诊断

### 文档与生成

- `docs/诊断规则与修复动作.md`
  - 诊断 code、分类、严重度、Quick Fix / CodeAction、说明
- `scripts/generateDiagnosticDocs.js`
  - 由 `diagnosticCodes` 与 `diagnosticActions` 生成诊断文档
  - `--check` 模式比较生成内容与仓库文件，不写入磁盘
- `scripts/checkReleaseConsistency.js`
  - 常规模式检查 package 与 lockfile
  - `--tag vX.Y.Z` 模式检查 README、CHANGELOG 与发布 tag
- `scripts/checkVsixContents.js`
  - 严格校验 VSIX 必须包含与禁止包含的文件
- `scripts/reportVsixArtifact.js`
  - 输出 VSIX 字节数与 SHA-256
- `.github/workflows/ci.yml`
  - main 分支与 Pull Request 自动运行测试、文档同步检查、ESLint 和 VS Code 集成测试
  - 集成测试失败时上传 Extension Host 日志
- `docs/macro-knowledge/README.md`
  - 当前 MACRO 知识、证据、规划和专题资料包的统一入口

## 发布流程

发布前固定执行：

```powershell
npm run docs:diagnostics
npm run docs:diagnostics:check
npm test
npm run test:integration
npm run package
```

然后更新：

- `package.json`
- `package-lock.json`
- `README.md` version badge
- `CHANGELOG.md`

提交并推送：

```powershell
git status --short
git add <files>
git commit -m "<message>"
git push origin main
```

创建 Release：

```powershell
gh release create vX.Y.Z .\syntec-macro-X.Y.Z.vsix --title "vX.Y.Z" --notes "<release notes>" --latest
```

确认：

```powershell
gh release list --limit 3
git status --short
git ls-remote --tags origin vX.Y.Z
```

## 注意事项

- 不要把旧版 `.vsix` 留在仓库根目录，避免误装。
- `test-demo.nc` 应保持无诊断，用作合法语法覆盖样例。
- 错误场景应放在测试内联样例中，不应长期留在 demo 文件里。
- Quick Fix 只做安全机械修复；需要用户判断语义的场景使用说明型 action。
- 动态跳转目标不做静态解析，避免误跳。
