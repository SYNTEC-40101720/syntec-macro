# 新代 MACRO 语法规范手册

本文档用于沉淀新代控制器 MACRO / LTP 机器人语法规范，作为 VS Code 插件的语法真源草案。

本文档目标：

- 记录已确认语法，减少规则散落在代码、snippet、README、测试和范例中的漂移。
- 区分正式语法、兼容但不推荐语法、不支持语法、待确认语法。
- 为 hover、snippet、diagnostics、TextMate grammar、测试用例提供统一依据。

## 0. 状态标记

| 标记 | 含义 | 插件策略 |
|---|---|---|
| 已确认 | 已由官方文档、实机或项目规则确认 | 可用于 hover、snippet、validator、测试 |
| 兼容不推荐 | 控制器可能接受，但不建议新代码生成 | validator warning |
| 不支持 | 控制器不支持或项目已确认不可用 | validator error |
| 待确认 | 资料不足或版本差异未确认 | 暂不做强诊断，写入待确认清单 |

## 1. 基础文件格式

### 1.1 MACRO 格式判定

状态：已确认

```iecst
%@MACRO
```

规则：

- MACRO 格式由文件内容决定，而非后缀名决定。
- 文件第一行为 `%@MACRO` 时，控制器核心按 MACRO 格式处理。
- 若无 `%@MACRO`，控制器核心按 ISO 格式处理。
- 若文件首个有效行是 `%` 而不是 `%@MACRO`，插件提示 warning：该文件可能被视为 ISO 格式。

### 1.2 文件后缀与加工档选择

状态：已确认

系统人机默认可选取的加工档后缀来自 `MMIConfig_Customer.xml` 的 Filter 配置。默认包含：

| 后缀 | 说明 |
|---|---|
| 无后缀 | 扩充 G 码与 O 码副程序常用形式 |
| `.nc` | 常用数控加工档 |
| `.cnc` | CNC 加工档 |
| `.tap` | 常见 CAM 输出 |
| `.prt` | 零件程序 |
| `.mpf` | 西门子风格主程序 |
| `.ptp` | 点到点加工程序 |
| `.pim` `.anc` `.bj` `.edit` `.demo` | 人机默认 Filter 可选后缀 |

默认 Filter：

```xml
<Filter>**.edit;**.demo;**.bj;**.anc;**.pim;**.tap;**.prt;**.mpf;**.ptp;**.cnc;**.nc;**.</Filter>
```

说明：

- 最后的 `**.` 代表允许选取无后缀文件。
- Filter 可由机械厂自行修改。
- 插件自动关联常见加工档后缀，但 `.txt` 不默认关联，避免抢占普通文本文件；`.txt` 若首行为 `%@MACRO` 仍可通过首行识别或手动切换语言处理。

### 1.3 扩充 G 码宏程序命名

状态：已确认

规则：

- 命名范围：`G200` ~ `G999`。
- 文件名不可以带后缀名。
- 支持 4 字命名与 6 字命名。

| 调用语法 | 文件名 |
|---|---|
| `G200` | `G0200` |
| `G200.1` | `G200001` |
| `G200.10` | `G200010` |

示例：

```text
正确：G0200
错误：G0200.NC
错误：G0200.TXT
```

### 1.4 O 码副程序命名

状态：已确认

规则：

- O 码副程序命名为 `O` + 数字，例如 `O8000`、`O0010`。
- O 码副程序同样使用无后缀文件名。
- 副程序呼叫时按文档管理的资料夹设定查找。

### 1.5 文件存放路径与优先级

状态：已确认

| 位置 | 路径 | 说明 |
|---|---|---|
| 加工档案 | `DiskA:\OpenCNC\NCFILES` | 一般加工程序；可在人机编辑 |
| 系统 MACRO | `DiskC:\OpenCNC\MACRO` | 系统级宏程序；人机中不可编辑 |
| 模拟器 | `DiskC\OpenCNC\MACRO` | 模拟器环境 |

查找优先级：

```text
NCFILES > MACRO
```

两处都有同名文件时，NCFILES 优先。

### 1.6 Macro 转换工具输入格式

状态：已确认

规则：

- 转换工具输入文件可为 `.txt` 或 `.nc`。
- 编码可为 UTF-8 或 BIG5/CP950。
- 这是转换工具输入格式，不代表控制器核心以 `.txt` 后缀判定 MACRO 格式。

### 1.7 插件识别策略

状态：已确认

| 场景 | 插件策略 |
|---|---|
| 常见加工档后缀 | 自动关联 `.nc`、`.cnc`、`.tap`、`.prt`、`.mpf`、`.ptp`、`.pim`、`.anc`、`.bj`、`.edit`、`.demo` |
| 任意首行为 `%@MACRO` 的文件 | 通过 firstLine 规则识别为 `syntec-macro` |
| `.txt` 文件 | 不默认关联，避免影响普通文本；可通过 `%@MACRO` 或手动切换语言识别 |
| `.macro` / `.scp` / `.G` | 未在默认 Filter 中确认，不默认关联；可通过 `%@MACRO` 或手动切换语言识别 |
| 无后缀扩充 G 码 / O 码 | 依首行 `%@MACRO` 识别；跳转搜索优先查无后缀文件 |

### 1.10 MACRO 读取/处理流程

状态：已确认 / 部分版本差异

MACRO 执行涉及三个阶段：档案识别、核心解译、运动执行。控制器核心会以预解方式提前解译 MACRO 内容，因此变量运算通常会超前于实际 G/M 码运动执行。

```text
档案识别 -> 核心解译/预解 -> 运动执行
首行 %@MACRO  逐行解译与建表  G/M 码实际驱动轴向运动
```

格式识别：

| 条件 | 处理方式 |
|---|---|
| 第一行为 `%@MACRO` | 视为 MACRO 格式，支援完整 MACRO 语法 |
| 第一行不含 `@MACRO` | 视为 ISO 格式，不支援 MACRO 语法 |

档案查找路径：

| 优先级 | 路径 | 说明 |
|---|---|---|
| 1 | `DiskA:\OpenCNC\NCFILES` | 一般加工档案资料夹 |
| 2 | `DiskC:\OpenCNC\MACRO` | 系统 MACRO 资料夹 |
| 3 | `DiskC:\OpenCNC\Res\*.res` | 打包的 res 文件内 |

若 NCFILES 与 MACRO 资料夹内存在同名文件，NCFILES 优先。APP Macro 独立路径为 `DiskC:\app\%AppName%\bin\Macro\`，只有该路径下的 Macro 才能存取 AR/MAR。

单节解译顺序：

| 顺序 | 内容 | 说明 |
|---|---|---|
| 1 | 模态 G 码 / 扩充 G 码宏程序 | 例如 `G15`、`G17`、`G70`、`G73`、`G84` |
| 2 | M 码宏程序 / T 码宏程序 | 已登录 M 码宏程序、`Pr3215=2` T 码宏程序 |
| 3 | S 码 | 主轴转速 |
| 4 | F 码 | 进给速度 |
| 5 | H 码 | 刀具补偿功能 |
| 6 | D 码 | 刀具补偿功能 |
| 7 | T 码 | 刀具选择，非宏程序时作为辅助码 |
| 8 | M 码 | 辅助码，非宏程序时作为辅助码 |
| 9 | B 码 | B 轴命令 |
| 10 | 插值 G 码 / 功能 G 码 | 例如 `G0`、`G1`、`G4`、`G51`、`G68` |

同一行若同时存在 G 码宏程序与 M 码宏程序，核心会先解译 G 码宏程序。若 G 码宏程序读取并占用了轴向引数，则同一单节中后续 M 码宏程序或插值 G 码可能读取不到这些引数。

预解机制：

- 变量运算在解译阶段即时完成。
- G/M 码运动在执行阶段才实际动作。
- 因此变量读取可能早于运动完成，读取机械座标或状态时容易取得旧值。
- `WAIT()` 可阻止预解继续进行，直到 WAIT 前的 G/M 码执行完毕。
- `M98`、`M99`、`M198` 不受 `WAIT()` 影响，`WAIT()` 不会等待它们执行完毕。

```iecst
%@MACRO
G01 X100. F500;
WAIT();
#1 := #1321;
G01 X(#1 + 50.);
M30;
```

流程跳跃资讯表：

- 控制器处理 `IF`、`CASE`、`WHILE`、`FOR`、`REPEAT` 等范围语法时，需要建立流程跳跃资讯表。
- 目前采用动态产生方式，边解译边建表。
- 动态建表可直接 CycleStart，不需等待完整扫描。
- 大档案中语法区间过大时，跨页跳转可能造成轴向降速。

大档案处理：

| 项目 | 规格 |
|---|---|
| 60KB 以内 | 完整支援所有语法 |
| 60KB 以上，10.120.32 及以后 | 支援 `IF` / `CASE` / `REPEAT` / `FOR` / `WHILE` 跨页 |
| 60KB 以上，10.120.32 之前 | 不支援有范围区间语法，可能触发 `COR-204` 档案太大 |

相关诊断变量：

| 诊断变量 | 说明 |
|---|---|
| `D320` | 解译总线单节数，反映解译器缓冲深度 |
| `D376` | 运动规划减速次数，持续增加时可能是语法区间过大 |
| `D978` | MacroExecutor 执行实际时间，平均值，单位 us |
| `D979` | MacroExecutor 执行实际时间，最大值，单位 us |
| `D994` | Script Executor 执行时间，平均值，Linux 支援 |
| `D995` | Script Executor 执行时间，最大值，Linux 支援 |

若 `D978` / `D979` 大于 4 倍 PLC 扫描时间，系统可能效能不足。Linux 系统支援上述执行线程诊断变量；Windows 系统预设为 0。

宏程序呼叫流程：

1. 父程序呼叫扩充 G 码、G65、M 码宏程序或副程序。
2. 控制器依路径优先级查找目标文件。
3. 对宏程序呼叫创建独立 `#1` ~ `#400` 区域变量空间；副程序呼叫依规则继承主程序变量空间。
4. 将引数写入对应 `#1` ~ `#26`，扩充引数使用 `GETARG` 读取。
5. 执行目标 MACRO。
6. `M99` 返回并回收独立区域变量空间。

`#1000` 插值模式：

- `#1000` 记录当前插值模式。
- MACRO 返回后，最后插值模式会影响父程序后续单节的执行模式。
- 遇到 `G00`、`G01`、`G02`、`G03`、`G31`、`G33` 或 `#1000` 内容变动时，`#1000` 会被自动改写。
- 若宏程序将 `#1000` 设为自身扩充 G 码号，父程序后续无显式插值模式的单节可能再次触发该宏程序。

实务建议：

- 读取运动后的座标或状态前，先加入 `WAIT()`。
- MACRO 内改变 `G90/G91` 时，进入时备份 `#1004`，离开前用 `G#10` 还原。
- 避免在 MACRO 中任意改变座标系统，例如 `G92`、`G54`、`G52`，以免影响父程序或图形模拟。
- MACRO 结尾应使用 `M99` 返回父程序。
- 长度或角度引数运算前建议使用 `STD` 标准化。
- 跨 MACRO 资料交换优先考虑模态变量 `#2001` ~ `#3100`，减少公用变量资源占用。
- 循环中加入 `SLEEP()` 可降低无穷循环锁死人机的风险。

## 2. 变量系统

### 2.1 区域变量 `#`

状态：已确认

```iecst
#1 := 100;
#[10 + #3] := 200;
```

基本范围：

| 编号 | 说明 | 规则 | 型态 |
|---|---|---|---|
| `#0` | VACANT 空白特殊值 | 只读；表示未赋值或未传入 | - |
| `#1` ~ `#26` | 系统内定宏程序引数区域变量 | 读写；宏呼叫引数自动写入 | Double |
| `#27` ~ `#400` | 宏程序与副程序使用的一般区域变量 | 读写；用于暂存、计算、内部流程 | Double |

规则：

- 区域变量使用数字编号，可用固定编号 `#1` 或表达式索引 `#[表达式]`。
- `#1` ~ `#400` 共 400 个可用区域变量；`#0` 另作为 VACANT 常数。
- 区域变量初始状态为 VACANT，MACRO 执行完毕离开时自动恢复 VACANT。
- `#TEMP` 这类命名变量不支持，插件报 error。
- 推荐赋值运算符为 `:=`。
- 赋值使用 `=` 可识别但不推荐，插件 warning。

### 2.2 宏程序引数映射

状态：已确认

当通过扩充 G 码、`G65`、`G66`、`G66.1`、M 码宏程序等方式呼叫宏程序时，传入引数会自动映射到 `#1` ~ `#26`：

| 引数 | 区域变量 | 引数 | 区域变量 | 引数 | 区域变量 |
|---|---|---|---|---|---|
| `A` | `#1` | `J` | `#5` | `S` | `#19` |
| `B` | `#2` | `K` | `#6` | `T` | `#20` |
| `C` | `#3` | `L` | `#12` | `U` | `#21` |
| `D` | `#7` | `M` | `#13` | `V` | `#22` |
| `E` | `#8` | `N` | - | `W` | `#23` |
| `F` | `#9` | `O` | - | `X` | `#24` |
| `G` | - | `P` | `#16` | `Y` | `#25` |
| `H` | `#11` | `Q` | `#17` | `Z` | `#26` |
| `I` | `#4` | `R` | `#18` |  |  |

规则：

- `G`、`N`、`O` 为控制器语法关键字，无法作为宏程序引数，没有对应的 `#` 变量。
- `#10`、`#14`、`#15` 没有引数字母对应，可作为一般区域变量使用。
- `L` 为特殊引数，用于设定宏程序连续执行次数；未提供时系统自动补 `L=1`。
- 若同一指令中某地址已有固定语义，例如 `P` 作为程序号，应以该指令语义为准。

引数分类：

| 类型 | 包含引数 | 特点 |
|---|---|---|
| 轴向引数 | `X` `Y` `Z` `A` `B` `C` `I` `J` `K` `U` `V` `W` | 只要其中一个被某宏程序占用，其他轴向引数均一起被占用 |
| 条件引数 | `F` `S` `T` `D` `E` `H` `P` `Q` `R` `M` `B` | 彼此独立，不会因任一被占用而连带 |
| 特殊引数 | `L` | 设定宏程序连续执行次数，未提供时系统自动补 `L=1` |
| 例外引数 | `G` `N` `O` | 控制器关键字，无法作为引数 |

扩充引数：

- `A`、`B`、`C`、`F`、`S`、`U`、`V`、`W`、`X`、`Y`、`Z` 可在字母后附带数字形成扩充引数，例如 `X1=50.`、`Z1=30.`。
- 扩充引数不直接映射到 `#1` ~ `#26`，需在宏程序内使用 `GETARG` 读取。
- `D`、`E`、`H`、`I`、`J`、`K`、`L`、`M`、`P`、`Q`、`R`、`T` 不可附带数字，例如 `D1=50.` 会报警。

```iecst
// 父程序
G200 X1=50. Y1=30.;

// G0200 宏程序内部
#27 := GETARG(X1);
#28 := GETARG(Y1);
```

### 2.3 区域变量生命周期

状态：已确认

区域变量的生命周期仅限该 MACRO 执行期间。执行完毕离开时，该区域变量自动恢复成 VACANT 状态。

| 呼叫方式 | 区域变量行为 | 生命周期结束时机 |
|---|---|---|
| `M98 P_` 副程序呼叫 | 继承主程序的 `#1` ~ `#400`，共用同一块变量 | 主程序执行完毕时 |
| `M198 P_` 副程序呼叫，未被登录时 | 继承主程序的 `#1` ~ `#400` | 主程序执行完毕时 |
| `G65 P_` 非模态宏程序呼叫 | 创建独立的 `#1` ~ `#400`，引数写入 `#1` ~ `#26` | 宏程序 `M99` 返回时 |
| `G66 P_` 模态宏程序呼叫 | 创建独立的 `#1` ~ `#400`，从 `G66` 到 `G67` 期间共用 | 执行 `G67` 后回收清除 |
| `G66.1 P_` 每单节模态宏程序呼叫 | 与 `G66` 相同 | 执行 `G67` 后回收清除 |
| 扩充 G 码宏程序 | 每次呼叫都创建新的 `#1` ~ `#400` | 该 MACRO 结束时恢复上一层 |
| M 码宏程序 | 每次呼叫都创建新的 `#1` ~ `#400` | 该 MACRO 结束时恢复上一层 |
| T 码宏程序，`Pr3215=2` | 每次呼叫都创建新的 `#1` ~ `#400` | 该 MACRO 结束时恢复上一层 |
| T 码副程序，`Pr3215=1` | 继承主程序的 `#1` ~ `#400` | 主程序执行完毕时 |

示例：

```iecst
// M98 继承：O8000 可读写主程序 #1
%@MACRO
#1 := 100;
M98 P8000;
M30;

// G65 独立：O8000 内 #1 来自 A，引数不影响主程序 #1
%@MACRO
#1 := 100;
G65 P8000 A50.;
M30;
```

### 2.4 VACANT `#0`

状态：已确认

规则：

- `#0` 永远为 VACANT，只读。
- 区域变量初始状态均为 VACANT。
- VACANT 不等于 `0`，表示未定义或未传入。
- 可用 `#0` 判断引数是否被传入。

```iecst
IF #24 = #0 THEN
  // X 引数未被传入
ELSE
  // X 引数已被传入
END_IF;
```

### 2.5 公用变量 `@`

状态：已确认 / 部分版本差异

```iecst
@1 := 100;
@[#3] := 567;
```

基本范围：

| 编号 | 说明 | 规则 | 型态 | 断电记忆 |
|---|---|---|---|---|
| `@0` | VACANT 空白特殊值 | 只读；表示未赋值 | - | - |
| `@1` ~ `@400` | 一般运算用变量 | 读写 | Double | 默认无；可通过 `Pr3811` 设定保留 |
| `@401` ~ `@655` | 对应 PLC 寄存器 `R1` ~ `R255` | 读写 | Long | 依 R 寄存器性质 |
| `@656` ~ `@999` | 有记忆性变量 | 读写 | Double | 有 |
| `@1000` | 旧版系统变量，请勿使用；新版改为有记忆性变量 | 旧版只读；新版读写 | Double | 版本相关 |
| `@1001` ~ `@1999` | 有记忆性变量 | 读写 | Double | 有 |
| `@10000` ~ `@14095` | 对应 PLC 寄存器 `R0` ~ `R4095` | 读写 | Long | 依 R 寄存器性质 |
| `@60000` ~ `@79999` | 扩充公用变量，需配合 `Pr3813`，仅适用 CE 系统 | 读写 | Double | 依配置 |
| `@100000` ~ `@165535` | 对应 PLC 寄存器 `R0` ~ `R65535` | 读写 | Long | 依 R 寄存器性质 |

生命周期：

- 公用变量使用数字编号，可用固定编号 `@1` 或表达式索引 `@[表达式]`。
- `@0` 永远为 VACANT，只读；VACANT 不等于 `0`，表示未定义。
- 公用变量范围为 `@0` ~ `@165535`，所有公用变量除非遇到关机，否则没有生命周期限制。
- 公用变量可跨 MACRO、跨轴群读写；系统复位后仍保留。
- `@656` ~ `@999`、`@1001` ~ `@1999` 原生具备断电记忆。
- `@1` ~ `@400` 默认无断电记忆，可通过 `Pr3811` 设定保留。
- `@60000` ~ `@79999` 为 CE 系统专用扩充区段，需配合 `Pr3813` 使用。
- `@TEMP` 这类命名变量不支持，插件报 error。

R 寄存器对应：

| 公用变量范围 | 对应 R 寄存器 | 型态 | 映射公式 |
|---|---|---|---|
| `@401` ~ `@655` | `R1` ~ `R255` | Long | R 号 = @ 号 - 400 |
| `@10000` ~ `@14095` | `R0` ~ `R4095` | Long | R 号 = @ 号 - 10000 |
| `@100000` ~ `@165535` | `R0` ~ `R65535` | Long | R 号 = @ 号 - 100000 |

R 寄存器可写入范围：

| R 寄存器范围 | 说明 |
|---|---|
| `R50` ~ `R80` | 可写 |
| `R101` ~ `R511` | 可写 |
| `R1024` ~ `R4095` | 可写 |
| `R5800` ~ `R7999` | 可写 |
| `R10000` ~ `R10999` | 可写 |
| `R15000` ~ `R65535` | 可写 |

未列出的 R 寄存器范围，例如 `R0` ~ `R49`、`R81` ~ `R100`、`R512` ~ `R1023`，属于系统保留区段，写入可能导致不可预期行为。DOS 系统仅支援 `R0` ~ `R7999`；WinCE / Linux 系统支援完整 `R0` ~ `R65535`。

型态与赋值：

```iecst
@1 := 123;       // Long 型态
@1 := 123.0;     // Double 型态
@10 := "12";     // 字串写入时会先进行 ASCII 转码

#1 := 123;
@[#1] := 567;      // 等同 @123 := 567
@[#1 + 7] := 890;  // 等同 @130 := 890
```

规则：

- 不同型态变量做四则运算时，会强制转型为 Double 再计算。
- 两个 Long 做四则运算时，结果维持 Long。
- R 寄存器对应区段强制为 Long。
- `MOD` 仅适用于 Long 型态，对 Double 型态使用 `MOD` 会触发警报。
- 公用变量存入字符串时会进行 ASCII 转码，读取时需使用 `SCANTEXT` 函数还原字符串内容。

应用场景：

| 场景 | 建议区段 | 原因 |
|---|---|---|
| MACRO 内部暂存运算 | `@1` ~ `@400` | 通用运算区，无断电记忆负担 |
| 跨 MACRO 资料传递 | `@1` ~ `@400` 或模态变量 `#2001` ~ `#3100` | 公用变量全局可见；模态变量复位消失更安全 |
| 与 PLC 交换数据 | `@401` ~ `@655`、`@10000` ~ `@14095`、`@100000` ~ `@165535` | 直接映射 R 寄存器 |
| 需要断电保存的工艺参数 | `@656` ~ `@999`、`@1001` ~ `@1999` | 原生断电记忆 |
| 关机后仍需保存的计数器或统计值 | `@656` ~ `@999` | 断电记忆且无需额外参数配置 |
| XML 数据读入 `DBLOAD` | 例如 `@1200` 等自定义区段 | 可配合 Schema 文件指定存放位置 |

### 2.7 变量使用建议

状态：已确认

- MACRO 内优先使用区域变量 `#1` ~ `#400`，除非有跨 MACRO 需求才使用 `@`。
- 执行 MACRO 副程序时，若模式 G 码会被改变，例如 `G90/G91`、`G40/G41/G42`，应在进入 MACRO 时备份当前状态，离开前恢复。
- 对长度或角度引数，运算前建议使用 `STD` 函数将单位标准化。
- 控制器核心会预解 MACRO 内容，若变量读写需与 G/M 码动作同步，应在读写前加入 `WAIT()`。
- 多个 MACRO 间交换数据时，优先使用模态变量，例如 `#2001` ~ `#2100`、`#3001` ~ `#3100`，系统复位时消失，可节省变量资源。
- MACRO 需要内定初始值时，可利用使用者参数，例如 `#4001` ~ `#4100`、`#5001` ~ `#5100`。

插件策略：

- 固定编号与 `#[表达式]` 均识别为区域变量。
- 固定编号与 `@[表达式]` 均识别为公用变量。
- 命名变量 `#TEMP`、`@TEMP` 报 error。
- `#0 := ...`、`@0 := ...` 后续可加入 warning，提示 VACANT 不建议写入。
- 区域变量固定编号超过 `#400` 时，后续可依变量区间提示用途或风险；对表达式索引不做静态范围诊断。
- 对静态可判定的 R 寄存器映射保留区写入，后续可加入 warning，提示可能导致不可预期行为。

### 2.8 常见变量警报

状态：已确认 / 部分待确认

| 场景 | 警报 |
|---|---|
| 存取超出开放范围或当前应用不允许的变量 | `COR-016` 不合法的变数存取 |
| ISO 格式档中使用 MACRO 语法存取 `#` / `@` 变量 | `COM-003` 句法错误 |
| 非 APP Macro 存取 AR / MAR | `COR-016` |
| 不同型态，Long / Double 做 `MOD` 运算 | 报警；`MOD` 仅适用 Long |
| Long 与 Double 型态做 `MOD` | `COR-054` 资料型别不相容 |
| 存取超出范围的公用变量 | `COR-016` 不合法的变数存取 |
| 写入 R 寄存器系统保留区段 | 可能导致不可预期行为，未必有警报保护 |

### 2.9 AR / MAR

状态：已确认 / 部分版本差异

```iecst
AR1 := 789;
MAR1 := 789;
AR[#3] := 789;
MAR[#3] := 789;
```

基本概念：

AR 和 MAR 是 APP 架构下的专用变量，用于避免传统公用变量 `@` / PLC `R` 在多个 APP 或客制功能之间产生变量冲突。每个 APP 拥有独立的 AR/MAR 空间，彼此互不干扰。

| 项目 | AR | MAR |
|---|---|---|
| 全称 | APP 区域变量 | APP 永久变量 |
| 断电记忆 | 无，关机后消失 | 有，永久储存 |
| 储存位置 | 内存 | `DiskC\app\%AppName%\usr\MAR.dat` |
| 型态 | Double；后续版本支援字符串 | Double |
| 编号方式 | 0-based，例如 `AR0`、`AR1` | 0-based，例如 `MAR0`、`MAR1` |

数量规格：

- 数量由每个 APP 的 `app.acfg` 配置文件定义。
- AR 默认上限 128 个，扩充后最多 2048 个。
- MAR 默认上限 1024 个，扩充后最多 2048 个。
- 新版本支援 `InitValue` 属性，例如 `<AR InitValue="Vacant">48</AR>`、`<MAR InitValue="0">512</MAR>`。

```xml
<?xml version="1.0" encoding="utf-8"?>
<Configuration Ver="1">
  <Load>TRUE</Load>
  <AR InitValue="Vacant">48</AR>
  <MAR InitValue="0">512</MAR>
  <APPVer>21213adad</APPVer>
  <CNCVer>10.118.12E</CNCVer>
</Configuration>
```

存取限制：

- AR/MAR 仅能在 `DiskC\app\%AppName%\bin\Macro\` 路径下的 APP Macro 文件中存取。
- 在其他路径的加工程序中存取 AR/MAR，会触发 `COR-016` 不合法的变数存取。
- AR/MAR 仅支援 MACRO 格式，第一行须有 `%@MACRO`；ISO 格式档中使用 AR/MAR 语法会触发 `COM-003` 句法错误。
- 每个 APP 的 AR/MAR 空间完全独立，APP A 的 Macro 无法直接存取 APP B 的 AR/MAR。
- 主程序无法直接存取任何 APP 的 AR/MAR，需通过呼叫 APP Macro 间接存取。

语法规格：

```iecst
%@MACRO
AR0 := 100;
AR46 := 72;
MAR53 := 10;
MAR0 := 3.14;

#1 := 10;
AR[#1] := 500;       // 等同 AR10 := 500
MAR[#1 + 5] := 200;  // 等同 MAR15 := 200

#27 := AR0;
#28 := MAR53;
@1 := AR5;
```

字符串规则：

- AR 后续版本支援储存字符串，每个变量可存 4 bytes。
- 写入较长字符串时会依 4 bytes 分段写入连续 AR 变量。
- 在 APP Macro 中，`SCANTEXT` 读取 AR 变量字符串；在一般 MACRO 中，`SCANTEXT` 读取公用变量 `@` 字符串。
- MAR 不支援储存字符串；对 MAR 赋值字符串时，程序内部会挡下，该变量值不受影响。

```iecst
AR100 := "ABCDEFGH";
// "ABCD" 存到 AR100
// "EFGH" 存到 AR101
// "\0"   存到 AR102

#30 := SCANTEXT(100);
```

生命周期：

| 事件 | AR | MAR |
|---|---|---|
| 开机 | 初始为 VACANT 或 0，依 `InitValue` 设定 | 从 `MAR.dat` 载入上次储存值 |
| 系统复位 | 保留，不清除 | 保留 |
| 关机 | 消失 | 自动储存到 `MAR.dat` |
| 加工中断电 | 消失 | 保留；`OP-023` 不支援检查 MAR 变量 |

APP 文件结构：

```text
DiskC\app\%AppName%\
  bin\
    app.acfg
    Macro\
      G0400
      G0500
    res\
      Schema\
      Screen\
  usr\
    MAR.dat
    CycData\
```

外部系统交互：

- APP Plugin 可通过 API 读写单个 AR/MAR、批量读写 AR/MAR，并取得 AR/MAR 容量。
- ORX 可直接读写 AR/MAR，与 APP Macro 交握。
- XMLDB Schema 支援 `AR1`、`MAR1` 等作为 `Storage` 来源。
- APP Macro 的 Modbus 语法支援将读取结果存入 AR/MAR。
- AR/MAR 可上传到云端平台，变量类别可区分为 `AR_%AppName%` 或 `MAR_%AppName%`。
- DS Logger 支援 AR 比较事件；事件引数仅支援整数型态，浮点数 AR 会四舍五入转型为整数处理。
- 人机画面 Display、VisibleRule、装置取值群组、图表元件可支援 AR/MAR 作为数据来源。

常见应用：

| 应用 | AR 用途 | MAR 用途 |
|---|---|---|
| 智慧进给 AFC | 操作状态交握、错误处理、示波器控制 | 多组参数、模式选择 |
| 品质追溯 | 画面选项、ACTION 控制 | 功能上云设定 |
| 暖机健检 | 检测结果暂存 | 历程数据永久储存 |
| 断刀回退 | 攻牙位置、转速、进给率等运动参数备份 | - |

常见警报：

| 场景 | 警报 |
|---|---|
| 非 APP Macro 路径中存取 AR/MAR | `COR-016` 不合法的变数存取 |
| 存取超出 `app.acfg` 定义范围的变量号码 | `COR-016` 不合法的变数存取 |
| 变量号码不是整数，例如 `MAR1.1` | 直接定址 `COM-003`；间接定址 `COR-016` |
| 变量号码为负数，例如 `MAR-1` | 直接定址 `COM-003`；间接定址 `COR-016` |
| ISO 格式档中使用 AR/MAR 语法 | `COM-003` 句法错误 |

插件策略：

- AR/MAR 为 APP 专用变量。
- 固定编号与表达式索引均识别为 APP 变量，例如 `AR1`、`MAR1`、`AR[#3]`、`MAR[#3]`。
- 静态可判定的负数或小数直接编号可报 error。
- 非 APP Macro 路径识别需结合工作区路径，后续可加入路径级 warning/error。
- 未读取 `app.acfg` 前，不应对 AR/MAR 最大编号做强诊断。

## 3. 运算子

### 3.1 运算子优先级

状态：已确认

数字越小，优先级越高。同一优先级的运算子从左到右依序执行。

| 优先级 | 运算子 | 符号 | 类别 |
|---|---|---|---|
| 1 | 括号 | `( )` `[ ]` | 分组 / 间接定址 |
| 2 | 函数赋值 | `Identifier(引数)` | 函数呼叫 |
| 3 | 负号 | `-` | 一元运算 |
| 3 | 补数 | `NOT` | 一元运算 |
| 4 | 乘号 | `*` | 算术运算 |
| 4 | 除号 | `/` | 算术运算 |
| 4 | 模数 | `MOD` | 算术运算 |
| 5 | 加号 | `+` | 算术运算 |
| 5 | 减号 | `-` | 算术运算 |
| 6 | 大小比较 | `<` `>` `<=` `>=` | 关系运算 |
| 7 | 等于 | `=` | 关系运算 |
| 8 | 不等于 | `<>` | 关系运算 |
| 9 | 布林且 | `&` `AND` | 逻辑运算 |
| 10 | 布林互斥 | `XOR` | 逻辑运算 |
| 11 | 布林或 | `OR` | 逻辑运算 |

### 3.2 赋值运算子

状态：已确认

| 符号 | 说明 | 范例 |
|---|---|---|
| `:=` | 变量赋值 | `#1 := 100;`、`@10 := 3.14;`、`AR0 := 50;`、`MAR1 := 200;` |

规则：

- MACRO 使用 `:=` 进行赋值。
- 单独的 `=` 是等于比较运算子，不是正式赋值运算子。
- 赋值使用 `=` 可识别但不推荐，插件 warning。

### 3.3 算术运算子

状态：已确认

| 符号 | 说明 | 范例 | 结果 |
|---|---|---|---|
| `+` | 加法 | `#1 := 10 + 20;` | `30` |
| `-` | 减法 | `#1 := 100 - 30;` | `70` |
| `*` | 乘法 | `#1 := 5 * 6;` | `30` |
| `/` | 除法 | `#1 := 10.0 / 3.0;` | `3.333...` |
| `MOD` | 取余数 | `#1 := 10 MOD 3;` | `1` |

一元负号：

```iecst
#1 := 10;
#2 := -#1;
#3 := -100;
```

### 3.4 除法

状态：已确认 / 部分待确认

规则：

- 使用 `/` 时，若分子与分母都是整数，结果仍为整数。
- 整数与非整数的差别在于有无小数点。
- 涉及除法时，建议至少在分子或分母加小数点，避免意外截断。

```iecst
#1 := 1. / 2;       // 结果 0.5
#2 := 1 / 2.0;      // 结果 0.5
#3 := 1 / 2;        // 结果 0，整数截断
#4 := (1/2) * 1.0;  // 先算 1/2=0，再乘 1.0，结果 0.0
```

不支持：

```iecst
#1 := 100 DIV 7;
```

插件策略：

- `DIV` 报 error。
- hover 说明 `/` 的整数除法规则。

待确认：

- 负数整数除法取整方向。
- 除以 0 的实际警报或行为。
- Long 纯运算溢出行为。

### 3.5 MOD

状态：已确认 / 待落地

规则：

- `MOD` 仅适用于 Long 型态。
- 任一操作数为 Double 型态时会触发 `COR-054` 资料型别不相容。
- 静态数值带小数点视为 Double。

```iecst
#1 := 10 MOD 3;    // 合法，结果 1
#2 := 10. MOD 3;   // 不合法，Double 不可使用 MOD
#3 := 10 MOD 3.0;  // 不合法，Double 不可使用 MOD
```

插件策略：

- 静态数字操作数带小数点时 warning/error。
- 变量参与时不静态判断。

### 3.6 关系运算子

状态：已确认

| 符号 | 说明 | 范例 |
|---|---|---|
| `=` | 等于 | `IF #1 = 10 THEN` |
| `<>` | 不等于 | `IF #1 <> 0 THEN` |
| `<` | 小于 | `IF #1 < 100 THEN` |
| `>` | 大于 | `IF #1 > 0 THEN` |
| `<=` | 小于等于 | `IF #1 <= 50 THEN` |
| `>=` | 大于等于 | `IF #1 >= 1 THEN` |

不支持：

```iecst
#1 == 100;
```

VACANT 比较：

```iecst
IF #1 = #0 THEN
  // #1 未赋值
END_IF;

IF @100 = @0 THEN
  // @100 未赋值
END_IF;
```

规则：

- VACANT 不等于 `0`。
- VACANT 表示未定义，`0` 是确定数值。

插件策略：

- `==` 报 error，提示使用 `=`。

### 3.7 逻辑运算子

状态：已确认

| 符号 | 说明 | 范例 |
|---|---|---|
| `&` `AND` | 逻辑且 | `IF (#1 > 0) & (#2 < 100) THEN` |
| `OR` | 逻辑或 | `IF (#1 = 1) OR (#1 = 2) THEN` |
| `XOR` | 逻辑互斥或 | `IF #1 XOR #2 THEN` |

规则：

- `&` 等同 `AND`。
- `NOT` 是一元数值补数，优先级见 3.1，不作为布林逻辑非推荐写法。
- 符号型运算符也应提供 hover。

取反推荐写法：

| 场景 | 推荐写法 | 范例 |
|---|---|---|
| 布林 0/1 翻转 | `1 - #变量` | `#100 := 1 - #100;` |
| 条件式布林取反 | 使用相反条件或与真值互斥 | `IF (#100 = 0) XOR (1 = 1) THEN` |
| 单 bit 翻转 | `XOR 1` 或对应 bit mask | `#100 := #100 XOR 1;` |
| 16 位整字翻转 | `XOR 0xFFFF`，mask 依实际字宽调整 | `#100 := #100 XOR 0xFFFF;` |
| 数值补数 | `NOT 数值`，结果为 `-(x + 1)` | `#2 := NOT #1;` |

```iecst
IF (#14 <= #12) AND (#15 <= #13) THEN
  #1 := 1;
END_IF;

REPEAT
  #14 := #14 + 1;
UNTIL (#14 > #12) OR (#15 > #13) END_REPEAT;

IF ((#14 = 4) & (#16 = 1)) THEN
  EXIT;
END_IF;
```

### 3.8 括号运算子

状态：已确认

| 符号 | 用途 | 范例 |
|---|---|---|
| `( )` | 运算分组 / 函数引数 | `#1 := (10 + 20) * 3;`、`SIN(45.0)` |
| `[ ]` | 间接变量定址 | `@[#1] := 500;`、`AR[#1 + 5] := 200;` |

### 3.9 型态转换规则

状态：已确认

| 运算情况 | 结果型态 | 范例 |
|---|---|---|
| Long 运算 Long | Long | `@1 := 2 * 100;` |
| Long 运算 Double | Double | `@1 := 1 + 2.0;` |
| Double 运算 Double | Double | `@1 := 1.5 + 2.5;` |
| Long `MOD` Long | Long | `@1 := 10 MOD 3;` |

### 3.10 运算子常见应用

状态：已确认

```iecst
IF #1 = 1 THEN
  #2 := 1;
ELSEIF (#1 >= 2) AND (#1 <= 5) THEN
  #2 := 2;
```

### 3.12 实用建议

状态：已确认

- 除法建议写成 `1.0 / 2`，避免 `1 / 2` 整数截断。
- 使用 `MOD` 前确认两侧均为 Long 型态，即静态数值不带小数点。
- VACANT 判断使用 `#0` / `@0`，例如 `IF #1 = #0 THEN`。
- 复合条件建议加括号，例如 `IF ((#1 > 0) & (#2 < 100)) THEN`。
- 赋值用 `:=`，比较用 `=`，两者不可混淆。

### 4.1 语法总览

### 4.2 变量指定

状态：已确认

```iecst
<变量> := <叙述>;
```

规则：

- 直接定值支援区域变量、公用变量、APP 变量与 APP 永久变量。
- 间接定值使用中括号 `[ ]`，不是小括号 `( )`。
- 字符串写入不同变量区时会依变量类型产生不同储存行为。

```iecst
@1 := 123;
#1 := 456;
AR1 := 789;
MAR1 := 789;

@10 := "12";  // 公用变量 @10 内容为 ASCII 转码值

#1 := 123;
@[#1] := 567;      // 等同 @123 := 567
@[#1 + 7] := 890;  // 等同 @130 := 890
AR[#1] := 789;     // 等同 AR123 := 789
MAR[#1] := 789;    // 等同 MAR123 := 789
```

### 4.3 GOTO

状态：已确认

```iecst
GOTO n;
```

规则：

- `GOTO` 的目标是 `N` 行号，不是程序行数。
- 可使用变量作为跳转目标，例如 `GOTO #2;`。
- 若程序中存在两个相同 `N` 行号，以第一个为准。
- 使用 `GOTO` 时应避免无穷循环；必要时在循环中加入 `SLEEP()`，让人机仍有机会中止程序。

```iecst
%@MACRO
#1 := 1;
#2 := 10;

IF #1 = 1 THEN
  GOTO #2;
END_IF;

IF #1 = 2 THEN
  GOTO 100;
END_IF;

N10;
G01 G90 X50. Y0. F1000;
M30;

N100;
G01 G90 X0. Y50. F1000;
M30;
```

### 4.8 FOR

状态：已确认

```iecst
FOR <变量> := <起始值> TO <终止值> BY <步进值> DO
  <陈述列表>
END_FOR;
```

规则：

- `ENDFOR` 可兼容识别，但建议使用 `END_FOR`。

规则：

- `Pr3601` ~ `Pr3610` 设为 `0` 时，不呼叫 `Mxxxx` 宏程序，直接进入 PLC 触发动作。
- 参数非 `0` 时，先呼叫对应 `Mxxxx` 宏程序，再依据宏程序内容决定是否进入 PLC 触发动作。
- 文件名为 `M` + 4 位数字，例如 `M123` -> `M0123`、`M50` -> `M0050`、`M1601` -> `M1601`。
- 文件名不可带后缀。
- 系统内定 M 码不可登录，包括 `M00`、`M01`、`M02`、`M30`、`M96`、`M97`、`M98`、`M99`。
- Pr3601 起登录的 M 码宏程序内，无法再呼叫其余 M 码或 T 码宏程序，而是直接触发 PLC 对应动作。
- 现况 Pr3601 起仅支援设定范围 `0` ~ `9999`，登录组数 10 组。
- 不可与 `Pr3599`、`Pr3600`、`Pr3804`、`Pr3742/3743`、`Pr3763` 等 M 码相关参数重复，否则触发 `OP-020` M 码相关参数设定冲突。

```iecst
Pr3601 = 123  // 登录 M123 -> M0123
Pr3602 = 124  // 登录 M124 -> M0124
Pr3603 = 50   // 登录 M50  -> M0050
```

M 码宏程序引数占用：

- M 码宏程序执行完毕后，会占用除 T 码以外的所有引数。
- 即使宏程序内只读取部分引数，执行完毕后所有引数，除 T 外，均被视为已占用。
- M 码作为宏程序或引数时，`#13` 会更新，但 R 值不更新；M 码作为辅助码时，`#13` 与 R 值均更新。

### 6.0 路径扩充引数

状态：已确认 / 待落地

控制器支援在路径单节后使用逗号形式的路径扩充引数，用于转角倒角、转角圆角、直线角度等 C-Type 语法。

```iecst
G01 X100. Y100., C10.;
G01 X100. Y100., R10.;
G01 X100. Y100., A45.;
```

规则：

- 目前确认支援 `,C_`、`,R_`、`,A_`。
- 不支援的路径扩充引数会触发 `COR-034` 路径扩充引数不存在。
- 逗号后的扩充引数不是一般轴地址引数，插件应独立识别。

错误示例：

```iecst
G91 G01 X100. Y100, Z100;
```

插件策略：

- 识别路径单节中的 `,C_`、`,R_`、`,A_` 为合法路径扩充引数。
- 对静态可判定的 `,Z_` 等未确认路径扩充引数报 error 或 warning，提示 `COR-034` 风险。

状态：已确认 / 部分跨行规则已落地

```iecst
MOVC X_ Y_ Z_ A_ B_ C_
     [A1=_ A2=_ A3=_ A4=_ A5=_ A6=_]
```

- `X/Y/Z/A/B/C/FL/FR/FEJ/PL/PQ/PR/ACC/DEC/P/Q` 为直接引数。

```iecst
MOVC [LP=_] [GP=_] X1=_ Y1=_ Z1=_ A1=_ B1=_ C1_=,
```

- Macro 标准语法，但会挡预解的指令除外，例如 `WAIT`、`READDI`、`READDO`。

- `M99`，副程序结尾。
- 模态指令：`FL/FR/FEJ`、`ACC/DEC`、`PL/PQ/PR`。

插件策略：

- `MOVC Xp=...`、`Yp=...`、`Zp=...` 报 error。
- `MOVC X=...`、`FL=...` 报 error。
- 单行 MOVC 未能与下一条 MOVC 配对时，插件报 error。
- 同行 `PL/PQ/PR` 出现两个以上时，插件报 error。

待确认：

- 两行参数是否必须同构。

- `P` 范围 0~1：0=急煞，1=软煞。
- 所有引数为直接引数。
- 跳脱后可读取系统变量：`#1441`~`#1458` 为跳脱时关节坐标反馈，`#1361`~`#1378` 为跳脱时程序坐标反馈，`#1608` 表示跳脱事件是否触发。
- 未设定 `SKIPCOND` 时，默认条件为 `C62 ON` 急煞。
- 不合法跳脱条件触发 `RBT-119`。

### 8.2 SWAITSIG

状态：已确认 / 部分已落地

```iecst
SWAITSIG P_ Q_ R_ [L_] [T_];
```

规则：

- `P` 范围 1~3：1=I-bit，2=R-bit，3=A-bit。
- `Q` 依 `P` 联动。
- `R` 范围 0~1。
- `L/T` 可省略，范围 0~2^31 ms。
- 运动单节后只能下 1 个 SWAITSIG。
- 核心会智慧判定减速等待讯号或加速通过并继续处理下一单节。
- 适用于连续运动、不减速到零的单节，例如有下 `PL/PQ/PR` 的移动单节。
- 替代语法为 `G04.101 E_ Q_ R_ C_ [T_] [P_]`。
- 等待多讯号不减速替代语法为 `G04.103`。
- 相关警报：`RBT-104`、`RBT-153`、`RBT-154-2`。
- 插件已检查：同一移动单节后超过 1 个 `SWAITSIG` 报 error；`WAIT()` 会切断此关系。

### 8.3 SYNCOUT

状态：已确认 / 部分已落地

```iecst
SYNCOUT S_ Q_ P_ R_ [L_] [K_];
```

规则：

- `S` 范围 1~3：1=O-bit，2=R-bit，3=A-bit。
- `Q` 依 `S` 联动。
- `P` 范围 0~100。
- `R` 范围 0~1。
- `L` 范围 0~10000 ms；`L0/L#0` 视为省略。
- `K` 范围 -10000~10000 ms；正数提前，负数延后。
- 单一有移动量移动单节最多 10 个 SYNCOUT。
- 核心功能为在移动单节中进行点位状态切换。
- 替代语法：运动中输出 `G11.103 E_ Q_ R_ P_ [T_]`；非运动中输出 `G11.101`。
- 不合法进阶输出触发 `RBT-110`。
- 插件已检查：同一移动单节后超过 10 个 `SYNCOUT` 报 error。

### 8.4 G10 L 指令

状态：已确认 / Hover 与 Snippet 已落地 / 静态范围诊断待落地

#### 8.4.1 G10 L1000 - R 寄存器写入

```iecst
G10 L1000 P_ R_;
```

功能：透过 G 码变更 R bit 的数值。

| 引数 | 说明 |
|---|---|
| `P` | R bit 号码 |
| `R` | 写入 R bit 的数值 |

规则：

- 在绝对指令模式（`G90`）下是绝对值，在增量指令模式（`G91`）下是增量值。
- 在 `G62` 模式下，指令会准确在 `G01/G02/G03` 单节交接处发出。
- 此指令之 `R` 值，会受到 `Pr17` 控制精度及 `Pr3241` 工件程序小数点型态影响。
- 如果 `R` 引数对应的变数具有物理意义，则单位为 LIU，会受到公英制影响。

范例：

```iecst
// Pr17=2
(G71 G90) G10 L1000 P1024 R10.1;   // 将 10.100 填入 R1024

// Pr3241=0
(G71 G90) G10 L1000 P1024 R10;     // 将 10 填入 R1024

// Pr3241=1、Pr17=2
(G71 G90) G10 L1000 P1024 R10;     // 将 10.000 填入 R1024

// Pr17=2，物理意义，公制
G71 G90 G10 L1000 P1024 R0.01;     // 表示 0.01 mm，将 10 填入 R1024

// Pr17=2，物理意义，英制
G70 G90 G10 L1000 P1024 R0.01;     // 表示 0.01 inch，将 100 填入 R1024
```

#### 8.4.2 G10 L1803 - MACRO IO TYPE-1 新格式

```iecst
G10 L1803 I_ Q_ P_ R_ [J_] [K_] [E_];
```

功能：在不减速的情况下，于运动单节内按距离触发讯号。支持设定维持时间、偏移时间和计算方向。

引数：

| 引数 | 说明 | 数值范围 |
|---|---|---|
| `I` | 讯号源 | 1=R-bit，2=O-bit，3=A-bit |
| `Q` | 讯号号码 | R-bit：0~65535 + 0~31（旧版 0~15）；O-bit：0~511；A-bit：0~511 |
| `P` | 单节触发距离（IU 线性距离） | >=0，不支援负值；从单节交接点处按 `E` 方向计算，超过单节长度则在起点或终点触发 |
| `R` | 讯号状态 | 0=LEVEL LOW，1=LEVEL HIGH |
| `J` | 讯号维持时间，可省略 | 0~3600000 ms；>0 时维持后输出相反极性，=0 或省略时输出一次 |
| `K` | 讯号偏移时间，可省略 | 单位 ms；正值=提早，负值=延迟，偏移无法超出该单节范围 |
| `E` | 触发距离计算方向，可省略 | 0=单节由前往后（默认），1=单节由后往前 |

规则：

- 核心差异：`G10 L1803` 在运动单节内按距离触发，`G10 L1805` 在单节间触发。
- 支持的运动指令：`G00`、`G01`、`G02`、`G03`、`G02.4`、`G03.4`、`G31`。
- 每个运动单节之间最多支持 12 组不重复的 `G10 L1803` 指令；超过触发 `COR-346`。12 组支持版本：10.118.41J、10.118.46。
- 引数超出范围触发 `COR-345`。
- 需接运动单节才会生效。
- 支持零移动单节，忽略 `P/K/E`，`J` 仍有效。
- 使用进给轴设定（`G10 L1100 P1002 R_`）时，触发距离参考进给轴合成移动量。
- `G31` 跳脱：若跳脱前已触发则不受影响，跳脱时未触发则该指令无效。
- `G02.4/G03.4` 时 `P` 引数可横跨两段圆弧，但 `K` 引数无法超出 `P` 所在圆弧段。
- 最低版本：10.118.28E、10.118.32 及之后。
- `G31` 支持版本：10.118.42S、10.118.48D、10.118.50 及之后。

#### 8.4.3 G10 L1805 - MACRO IO TYPE-2

```iecst
G10 L1805 I_ Q_ R_ [J_];
```

功能：在单节间（非运动过程中）触发讯号，触发前后可以不为移动单节。

引数：

| 引数 | 说明 | 数值范围 |
|---|---|---|
| `I` | 讯号源 | 1=R-bit，2=O-bit，3=A-bit |
| `Q` | 讯号号码 | R-bit：0~65535 + 0~31（旧版 0~15）；O-bit：0~511；A-bit：0~511 |
| `R` | 讯号状态 | 0=LEVEL LOW，1=LEVEL HIGH |
| `J` | 讯号维持时间，可省略 | 0~3600000 ms；>0 时维持后输出相反极性，=0 或省略时输出一次 |

规则：

- 确认讯号不与 PLC 冲突。
- 引数超出范围触发 `COR-345`。
- `G10 L1803` 与 `G10 L1805` 指令使用过多触发 `COR-346`。
- 可用 `G10 L1801` 取消正在输出的讯号，原状态维持不变。
- 可用 `G10 L1802` 等待讯号维持时间结束后再执行后续。
- 最低版本：10.118.28G、10.118.33 及之后。

#### 8.4.4 G10 L1803/L1805 共通限制与 Q 引数范例

Q 引数范例：

| 讯号 | 写法 |
|---|---|
| A310 | `I3 Q310` |
| R49 | `I1 Q4900` |
| R50.1 | `I1 Q5001` |
| O1 | `I2 Q1` |

共通限制：

- 不支援：手轮模拟、加工中调 Override、RTCP 且 Pr3054=1、Pr411=1 G00 独立运动、中断型副程序。
- `M30` / 重置 / 急停均会清除所有 R/O/A 点状态控制；暂停会暂停，Cycle Start 恢复。
- 10.118.40R、10.118.42R、10.118.48C、10.118.50 及之后版本静音模式不支援。

范例：

```iecst
G01 X10.;
G10 L1805 I2 Q50 R1 J1000;
G10 L1000 P1025 R200;
```

说明：X 走到 10 mm 时触发 O-50 ON，同时写入 R1025=200；O-50 触发 1 秒后返回原状态。

```iecst
G90 G00 X0;
G01 X10. F3000
G10 L1803 I2 Q50 P3. R1 J1000 K300;
G01 X20.;
M30;
```

说明：距上单节起点 3 mm，提前 300 ms，O50 ON 维持 1000 ms。

#### 8.4.5 G10 L1810 - 设定讯号条件

| `R` | 讯号状态 | 0=LEVEL LOW，1=LEVEL HIGH |

#### 8.4.6 G10 L1820 - 等待讯号条件成立

```iecst
G10 L1820 [P_] [K_];
```

功能：需搭配 `G10 L1810` 设定讯号条件，再由 `G10 L1820` 执行等待。

| 引数 | 说明 | 数值范围 |
|---|---|---|
| `P` | 走到上一单节终点后，讯号等待时间 | 0~3600000 ms |
| `K` | 逾时处理方式 | 0 或省略：发出警报 `COR-338`；1：不继续等待，执行下一单节 |

`P` 引数说明：

- 单位为 ms，只支援整数，不支援负值或小数。
- 不受执行时 Override 影响，例如 `P5000` 不管 Override 怎么改都是等 5 秒。
- 省略 `P`：走到单节终点后停下等待，直到讯号条件成立才继续执行，忽略 `K` 引数。
- `P=0`：走到单节终点后立刻执行 `K` 引数的逾时处理方式。
- `P>0`：走到单节终点停下，等待 `P` 时间后若条件仍未成立，执行 `K` 引数逾时处理。

规则：

- `G10 L1810/L1820` 须单独编写在一行，不要和移动单节编写在同一行。
- 设定完 `G10 L1810` 后，只有执行到 `G10 L1820` 才会进行等待。
- 执行完第一个 `G10 L1820` 后，若需执行第二个，须重新设定 `G10 L1810`，不能沿用之前的条件。
- 设定多个 `G10 L1810` 时，所有讯号条件都必须成立才算完成等待。
- 多个 `G10 L1810` 共用同一个 `G10 L1820` 的等待时间和逾时处理方式。
- `G10 L1810` 一定要设定 `I/Q/R` 引数，缺少任一触发 `COR-345`。
- `G10 L1810` 的 `I/Q/R/J` 引数超出范围、负值或小数触发 `COR-345`。
- 每个 `G10 L1820` 最多支援 5 个 `G10 L1810` 条件，超过触发 `COR-335`。
- `G10 L1820` 的 `P/K` 引数超出范围、负值或小数触发 `COR-345`。
- 加工档必须先编写过至少一个 `G10 L1810`，否则触发 `COR-337`。
- `G10 L1820` 必须编写在想要等待的单节下方，除 `G10 L1810` 外中间不可插入其他指令。
- 搭配轴向交迭（overlapping）使用时，若交迭距离超过且条件成立，移动单节会和下一单节交迭。
- 在轴向交迭单节后仅能使用一个 `G10 L1820`，不可重复编写。
- 急停、`M30`、重置会取消等待。
- 暂停或单节停止（`M00/C40`）会暂停等待，时间暂停计时不清除，重新执行后继续。
- 手轮模拟反转时无效；手轮模拟反转至该组任何 `G10 L1810` 或 `G10 L1820` 指令再正转，等待讯号条件皆成立。
- 不支援 `G05` 高精轨迹控制（HPCC）、平滑刀尖点控制（STCP）、高速预解功能。
- 不支援 `G53` 接 `G53` 轴向交迭中使用。
- `COR-334` 警报支援 10.118.34 及之前版本；10.118.35 及之后变更为 `COR-345`。
- 10.118.40R、10.118.42R、10.118.48C、10.118.50 及之后版本静音模式不支援。
- 支援版本：10.118.19 及之后。

范例：

```iecst
G00 X0. Y0.;
G00 X50.;
G10 L1810 I2 Q1 R0;       // 条件1：I1 = OFF
G10 L1810 I2 Q2 R1;       // 条件2：I2 = ON
G10 L1810 I2 Q3 R1;       // 条件3：I3 = ON
G10 L1820 P5000;          // 等待5秒，逾时跳警报
G00 Y50.;
M30;
```

```iecst
G00 X0.;
G10 L1000 P51 R1;         // 设定 R51 = 1
G10 L1810 I1 Q5100 R1 J100;  // 条件：R51.0 = ON
G10 L1820 P1000 K1;       // 等待1秒，逾时不跳警报继续执行
G00 X50.;
M30;
```

#### 8.4.7 G10 通讯相关指令

##### G10 L1021 - 发送 ENIP 通讯命令

```iecst
G10 L1021 [Q_] P_ S_ C_ I_ [A_] [X_] [Y_] [U_] [V_];
```

功能：发送 EtherNet/IP 通讯命令。

| 引数 | 说明 | 数值范围 |
|---|---|---|
| `Q` | 网卡号，可省略 | 省略或 0：自动搜寻所有网卡 |
| `P` | 站号，即设备 IP 最后一码 | 子网路遮罩须为 255.255.255.0 |
| `S` | Service 服务代码 | 0~255，十进制 |
| `C` | Class 类别 ID | 0~65535，十进制 |
| `I` | Instance 实例 ID | 0~2^31-1，十进制 |
| `A` | Attribute 属性 ID，可省略 | 0~65535，不需要时可省略或给 0 |
| `X` | Request Data 阵列起始 R 值位置 | 例如 `X20000` 代表从 R20000 开始 |
| `Y` | Request Data 阵列资料长度 | 每个 R 值=1 Byte，范围 0~255 |
| `U` | Response Data 阵列起始 R 值位置 | 起始 R 值的数值代表返回资料长度 |
| `V` | Response Data 阵列最大储存长度 | 须 >=1，含 `U` 指定的起始 R 值 |

规则：

- 若不同网域下有相同站号（如网卡 1 的 192.168.1.2 和网卡 2 的 192.168.10.2 都是第 2 站），需透过 `Q` 指定；省略 `Q` 则优先搜寻第一张网卡。
- Request Data 范例：传送 3 bytes ASCII `{'A','B','C'}` = `{65,66,67}`，从 R20000 开始写 `X20000 Y3`。
- Response Data 范例：规划 R20010~R20019 共 10 个 R 值，写 `U20010 V10`。回传 `{'D','E','F'}` = `{68,69,70}` 时，R20010=3，R20011=68，R20012=69，R20013=70。
- 使用前请先确认 EtherNet/IP 设备开机连线正常。
- Macro 引数格式或范围错误触发 `COR-350`；网卡号/站号错误或找不到设备触发 `OP-079` + `COR-351`；Service/Class/Instance/Attribute/Request Data 不支援触发 `COR-351`；Response Data 长度超过最大储存长度 `V` 触发 `COR-352`。
- 加工中手轮模拟时，手轮反转无法回退至此指令。
- 支援版本：10.118.22A、10.118.23 以上。

##### G10 L1022 - EtherCAT 物件字典读写功能

```iecst
G10 L1022 P0 A_ I_ U_ R_;
G10 L1022 P1 A_ I_ U_ B_ V_ E_;
```

模式：

| 模式 | 格式 |
|---|---|
| 读取 | `G10 L1022 P0 A_ I_ U_ R_` |
| 写入 | `G10 L1022 P1 A_ I_ U_ B_ V_ E_` |

引数：

| 引数 | 说明 |
|---|---|
| `P` | 0=读取，1=写入 |
| `A` | 轴号 |
| `I` | 物件 Index，支援十进制或十六进制，十六进制格式如 `"2000h"` |
| `U` | 物件 SubIndex，支援十进制或十六进制 |
| `R` | 读取结果存放的 R 值位置，读取模式使用 |
| `B` | 写入资料的 BitLength，限制为 8、16、24、32 |
| `V` | 写入的数值，支援十进制或十六进制 |
| `E` | 执行结果存放的 R 值位置，写入模式使用 |

规则：

- 读取结果：指定 `R` 值存放读取到的数值，`R+1` 存放 Error Code。写入结果：`E` 值存放 Error Code。
- Error Code：0=失败，1=成功，2=执行中，3=不支援此物件。
- 读取模式支援所有 ESI 有定义的物件；写入模式只支援部分 ESI 有定义的物件。
- `I_`、`U_`、`V_` 使用十六进制字串格式错误触发 `COR-023`。
- 除 `I_`、`U_`、`V_` 外，所有引数须以整数输入，否则触发 `COR-146`。
- 一次只执行一个物件读写命令，多轴群同时使用需排队等待；与 EtherCAT Debug Mode 同时触发时亦需排队。
- 写入成功后不会同步更新总线参数页面的数值。
- 执行时自行挡预解，减速至零后才做读写动作。
- 使用 `@` 变数读取结果时，需在读取前加入 `WAIT()`。
- 支援版本：10.118.40H、10.118.45 及以后版本。
- `B` 引数限制（8/16/24/32）自 10.120.32V、10.120.42J、10.120.51、12.0.7 起。

##### G10 L1900/L1901/L1910/L1911 - MODBUS 通讯指令

格式总览：

| 类型 | TCP | RS485 |
|---|---|---|
| 读取一至多笔 | `G10 L1900 C3 I_ A_ Q_ K_` | `G10 L1910 C3 I_ A_ Q_ K_` |
| 写一笔资料 | `G10 L1900 C6 I_ A_ X_` | `G10 L1910 C6 I_ A_ X_` |
| 自定义封包 | `G10 L1901 P_ R_ Q_ [K_]` | `G10 L1911 P_ R_ Q_ [K_]` |

所有引数皆为十进制。

引数：

| 指令 | 引数 | 说明 |
|---|---|---|
| 读取 `C3` | `C` | Function Code = 3 |
| 读取 `C3` | `I` | Slave 站号 |
| 读取 `C3` | `A` | Slave 装置资料读取起始位址 |
| 读取 `C3` | `Q` | 回传资料存放的起始 R 值 |
| 读取 `C3` | `K` | 回传资料的数量（读取数量） |
| 写入 `C6` | `C` | Function Code = 6 |
| 写入 `C6` | `I` | Slave 站号 |
| 写入 `C6` | `A` | Slave 装置资料写入位址 |
| 写入 `C6` | `X` | 写入的资料，范围 0~65535 |
| 自定义封包 | `P` | 自定义资料参考的起始 R 值 |
| 自定义封包 | `R` | 自定义资料参考的 R 值数量，不可超过 254 |
| 自定义封包 | `Q` | 回传资料存放的起始 R 值 |
| 自定义封包 | `K` | 回传资料的数量上限，可省略，省略等效 `K0`，不填入回传资料 |

执行流程与设定：

- 封包收送期间光标停在该行，收到回传资料并成功后才继续下一行。
- 通讯错误时跳警报并停在该行，所有轴群停止。
- 执行完毕成功后触发 R608.10。
- 用 `#` 或 `@` 变数读取回传值，需在指令后加 `WAIT()` 挡预解。
- RS485：设定 R5040 Modbus 通讯埠，SUPER/10/20/200 使用 R5040=3，eHMC/3/6/11/21 使用 R5040=2；并设定 Pr3941~Pr3950（COM2）或 Pr3961~Pr3970（COM3）。
- TCP：设定 Pr3971 Modbus TCP 通讯逾时异常时间。

注意事项：

- 请勿与 Modbus PLC 同时执行。
- 指令格式错误触发 `COR-356`；封包内容错误触发 `COR-357`；逾时且 Retry 后仍无回传触发 `COR-358`；回传封包异常触发 `COR-359`。
- 不用特别处理 R608.0，指令会自行触发传送。
- 逾时时间：RS485 为 Pr3950/Pr3970，TCP 为 Pr3971。
- 手轮反转无法回退至此指令。
- `Q` 引数指定的 R 值超过范围无法储存；`P` 引数超过范围使用预设值 0，可能导致 `COR-358`。
- 支援版本：10.118.28D、10.118.31 及以后。

### 8.5 STITCHON / STITCHOFF

状态：已确认 / 部分已落地

```iecst
STITCHON S_ Q_ [L_ / K_] E_;
STITCHOFF;
```

规则：

- `L` 与 `K` 二选一，只能择一输入。
- `L` 范围 1~10000 ms，仅整数，不可加小数点。
- `K` 范围 0.001~100 mm。
- `E` 范围 0.001~100 mm。
- 区间内存在禁用指令时可能报 RBT-115。
- 插件已检查：`L/K` 同时出现报 error；`L/K` 都省略报 warning；`L` 带小数点报 error。
- 插件已检查：`STITCHON`~`STITCHOFF` 区间内出现 `MOVJ`、`USERCOR`、`SHIFTON/OFF`、`OBJCORON/OFF/CLEAR`、`SYNCOUT`、`WEAVEON/OFF` 或带 `SKIP` 的 `MOVL/MOVC/INCMOVL` 报 error；`M96` 报 warning。
- `WAITSYNC/ENDSYNC` 也属于禁用项。
- 相关警报：`RBT-114`、`RBT-115`、`RBTWRN-010`。

### 8.6 WEAVEON / WEAVEOFF

状态：已确认 / 部分已落地

```iecst
WEAVEON P_;
WEAVEON E_ Q_ K_ L_ [R_] [I_];
WEAVEOFF;
```

规则：

- 两种语法互斥，不可混用。
- `P` 范围 1~50，整数。
- `E` 范围 0~50 mm。
- `Q` 范围 0.1~5 Hz，需小数形式，例如 `Q1.0`。
- `K` 范围 0~180 deg。
- `L` 范围 0~1000000 ms，不可加小数点。
- `R/I` 范围 0~1，不可加小数点。
- WEAVEON 区间内存在禁用指令时可能报 RBT-322。
- 插件已检查：`P_` 语法不可与 `E/Q/K/L/R/I` 混用。
- 插件已检查：`WEAVEON`~`WEAVEOFF` 区间内出现 `MOVJ` 或 `STITCHON/OFF` 报 error；`M96` 报 warning。
- `WAITSYNC/ENDSYNC` 与手轮模拟倒退也属于禁用项。
- 相关警报：`RBT-321`~`RBT-327`、`COR-064`、`RBTWRN-010`、`RBTCRI-011`。
- 插件已提示：细节语法中的 `Q` 频率建议使用小数形式，例如 `Q1.0`。

- `DRVDATA` 每次呼叫约 0.1~0.2 秒。
- 站号无对应驱动器时，`DRVDATA` 回传 VACANT。
- 开机一分钟内呼叫 `DRVDATA` 可能触发 `COR-016`。

## 10. MACRO 撰写注意事项

### 10.1 档案格式与命名

状态：已确认

| 项目 | 正确做法 | 风险 |
|---|---|---|
| 格式宣告 | 第一行使用 `%@MACRO` | 否则视为 ISO 格式，MACRO 语法触发 `COM-003` |
| 扩充 G 码 | 使用无后缀 `G0200`、`G200001` | 加后缀会导致控制器无法按扩充 G 码规则查找 |
| O 码副程序 | 使用无后缀 `O8000`、`O0010` | 加后缀会导致副程序查找不一致 |
| 一般加工档 | 使用 `.nc`、`.cnc`、`.tap` 等加工档后缀 | 后缀不决定 MACRO 格式，仍需看首行 |
| 系统 MACRO | 放在 `DiskC:\OpenCNC\MACRO` | 人机中不可编辑 |
| APP Macro | 放在 `DiskC:\app\%AppName%\bin\Macro\` | 只有该路径可存取 AR/MAR |

### 10.2 预解与 WAIT

状态：已确认

核心会预解 MACRO 内容，变量运算速度会超前于 G/M 码实际运动执行。读取运动后的座标、状态或系统变量前，应使用 `WAIT()` 同步。

```iecst
%@MACRO
G01 X100. F500;
WAIT();
#1 := #1321;
G01 X(#1 + 50.);
M30;
```

规则：

- `WAIT()` 可等待前方 G/M 码执行完成后再继续解译。
- `M98`、`M99`、`M198` 不受 `WAIT()` 影响。
- 运动后立即读取机械座标或状态时，省略 `WAIT()` 容易读到旧值。

### 10.3 模态备份与还原

状态：已确认

MACRO 内部若改变 `G90/G91`、`G40/G41/G42` 等模态 G 码，离开 MACRO 后不会自动还原，会影响父程序后续执行。

常用模态系统变量：

| 系统变量 | 对应模态 | 说明 |
|---|---|---|
| `#1000` | 插值模式 | `G00/G01/G02/G03` |
| `#1003` | 绝对/增量 | `G90/G91` |
| `#1004` | 绝对/增量 | `G90/G91` |
| `#1015` | 刀具补偿 | `G40/G41/G42` |
| `#1016` | 刀长补偿 | `G43/G44/G49` |

```iecst
%@MACRO
#10 := #1004;
#11 := #1015;

G91 G00 Y10.;
G41 D1;

G40;
G#10;
M99;
```

G 码宏程序返回前还原状态示例：

```iecst
// Main
G90;
G01 X10. F100.;  // X=10
G200;            // 呼叫 G0200，X 增量移动 10，执行后 X=20
X-20.;           // G0200 已还原 G90/G01，因此此行按主程序预期执行
M30;

// G0200
%@MACRO
#101 := #1000;   // 备份插值模式 G00/G01/G02/G03/G33/G34/G35
#102 := #1004;   // 备份 G90/G91
G91 G00 X10.;
G#101;           // 还原插值模式
G#102;           // 还原 G90/G91
M99;
```

### 10.5 引数处理

状态：已确认

规则：

- 长度或角度引数运算前建议使用 `STD` 标准化。

### 10.7 运算与型态陷阱

状态：已确认

```iecst
#1 := 1 / 2;         // 结果 0，整数除整数
#1 := 1. / 2;        // 结果 0.5
#1 := 1 / 2.0;       // 结果 0.5
#1 := (1/2) * 1.0;   // 结果 0.0
#1 := (1./2) * 1.0;  // 结果 0.5

#2 := 10 MOD 3;      // 合法
#2 := 10. MOD 3;     // COR-054 风险
```

规则：

- 除法至少让分子或分母带小数点，避免整数截断。
- `MOD` 仅适用于 Long 型态。
- 括号内表达式会先计算，括号内若发生整数截断，后续乘以 Double 也无法恢复精度。

### 10.8 循环与格式

状态：已确认

规则：

- `WHILE`、`REPEAT`、`FOR`、`GOTO` 都有无穷循环风险，循环中建议适时加入 `SLEEP()`。
- 不要在 `FOR` 循环内使用跳转进出循环的命令，包括往返 `GOTO`、复式切削循环 `G72` ~ `G78`、`M98 H_`。
- `IF/CASE/WHILE/FOR/REPEAT` 嵌套上限为 10 层，超过会触发 `COM-007`。
- 使用空格或 Tab 缩排，提高可读性。
- MACRO 格式注释使用 `//` 或 `(* ... *)`，不要与 ISO 的 `( ... )` 注释混用。

需要分号的语法：变量赋值、G/M 码指令、`GOTO`、`EXIT`、`END_IF`、`END_FOR` 等结尾语句。

不需要分号的语法：`IF ... THEN`、`ELSEIF ... THEN`、`ELSE`、`WHILE ... DO`、`FOR ... DO`、`REPEAT`、`CASE ... OF`、`N:` 条件标签行。

| 现象 | 可能原因 | 排查方向 |
|---|---|---|
| 变量值不正确 | 预解导致读取时机错误 | 加 `WAIT()` |
