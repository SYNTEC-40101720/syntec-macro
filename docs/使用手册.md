# 第一部分 安装与配置

## 1.1 安装方法

### 方法 1：从 VSIX 文件安装（推荐）

1. 下载 `syntec-macro-2.4.0.vsix`
2. 在 VS Code 中按 `Ctrl+Shift+P`
3. 输入 `Install from VSIX...`
4. 选择下载的 `.vsix` 文件
5. 重新加载窗口（`Ctrl+Shift+P` → `Reload Window`）

### 方法 2：命令行安装

```bash
code --install-extension syntec-macro-2.4.0.vsix
```

### 方法 3：从源代码构建

```bash
git clone https://github.com/SYNTEC-40101720/syntec-macro-vscode.git
cd syntec-macro-vscode
npm install
npm run package
code --install-extension syntec-macro-2.4.0.vsix
```

## 1.2 文件关联

扩展自动关联以下文件扩展名：

| 扩展名 | 说明 |
|--------|------|
| `.macro` | 宏程序文件（主要） |
| `.scp` | 脚本文件 |
| `.G` | G 代码程序文件 |
| `.nc` | NC 程序文件 |

此外，首行为 `%@MACRO` 的任何文件也会被自动识别。

## 1.3 配置选项

打开 VS Code 设置（`Ctrl+,`），搜索 `syntecMacro`：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `syntecMacro.enableDiagnostics` | boolean | `true` | 启用实时语法诊断 |
| `syntecMacro.enableCompletions` | boolean | `true` | 启用智能补全 |
| `syntecMacro.enableHover` | boolean | `true` | 启用悬停文档 |
| `syntecMacro.includePath` | array | `[]` | G65 宏程序文件额外搜索路径（绝对路径） |

### 编辑器默认设置

扩展为 `syntec-macro` 语言自动设定以下编辑器参数：

| 设置项 | 值 | 说明 |
|--------|-----|------|
| `editor.tabSize` | 4 | 缩进 4 空格 |
| `editor.insertSpaces` | false | 使用 Tab 缩进 |
| `editor.autoIndent` | advanced | 高级自动缩进 |
| `editor.colorDecorators` | false | 禁用颜色装饰器（防止 `#100` 等变量显示为色块） |
| `editor.defaultColorDecorators` | never | 禁用默认颜色装饰器 |

### includePath 配置示例

```json
{
  "syntecMacro.includePath": [
    "C:\\CNC\\MACROS",
    "D:\\PROJECTS\\COMMON",
    "\\\\SERVER\\SHARE\\MACROS"
  ]
}
```

## 3.2 高亮示例

```macro
%@MACRO                          (* 宏程序头 - 关键字 *)
#1 := 100;                       (* 局部变量 + 赋值 *)
@100 := 200;                     (* 全局变量 *)
// 单行注释                      (* 注释 *)
(* 块注释                        (* 多行 *)
   跨越多行 *)
IF #1 > 50 THEN                  (* 条件控制 *)
  #2 := ABS(#1 - 50);            (* 函数调用 *)
  MSG("Value: " + STR(#2));      (* 字符串 + 函数 *)
END_IF;
G00 X#1 Y@100;                   (* G 代码 + 变量混合 *)
MOVJ C1=0 C2=0 FJ=50 PL=5;       (* 机器人指令 *)
M99;                              (* 程序结束 *)
```

# 第四部分 智能补全

## 4.1 触发方式

| 触发条件 | 补全内容 |
|----------|----------|
| 输入 `#` | 常用局部变量 `#1` ~ `#20` + 大变量 `#100`, `#500`, `#1000`, `#2000`, `#9901`~`#9906` |
| 输入字母前缀（如 `ST`） | 匹配的函数名（`STR`, `STR2INT`）、关键字、G/M 代码 |
| 输入 `G` + 数字 | 匹配的 G 代码 |
| 输入 `M` + 数字 | 匹配的 M 代码 |
| `Ctrl+Space` | 手动触发所有补全 |

## 4.2 补全类型

### 函数补全

输入函数名前缀，自动弹出含签名的补全列表。选中后自动插入函数名和括号 snippet，光标定位到括号内。

| 输入 | 补全结果 | 说明 |
|------|----------|------|
| `AB` | `ABS(${1})` | 光标在括号内 |
| `SQ` | `SQRT(${1})` | 光标在括号内 |
| `ST` | `STR(${1})`, `STR2INT(${1})` | 多个匹配项 |

### 关键字补全

| 输入 | 补全结果 |
|------|----------|
| `IF` | `IF` |
| `FO` | `FOR` |
| `WH` | `WHILE` |
| `CA` | `CASE` |
| `RE` | `REPEAT` |
| `EL` | `ELSE`, `ELSEIF` |
| `EN` | `END_IF`, `END_FOR`, `END_WHILE`, `END_REPEAT`, `END_CASE` |
| `GO` | （不补全 GOTO，避免误用） |

### G 代码补全

输入 `G` 后跟数字，匹配所有已定义的 G 代码（共 80+ 个）。

### M 代码补全

输入 `M` 后跟数字，匹配所有 M 代码（`M00` ~ `M99`，共 100 个）。

### 变量补全

输入 `#` 后弹出：

| 变量 | 说明 |
|------|------|
| `#1` ~ `#20` | 常用局部变量 |
| `#100` | 大变量 |
| `#500` | 大变量 |
| `#1000` | 大变量 |
| `#2000` | 大变量 |
| `#9901` ~ `#9906` | 系统专用变量 |

## 4.3 补全设置

通过 `syntecMacro.enableCompletions` 配置项控制（默认 `true`）。

# 第六部分 代码跳转

## 6.1 GOTO 跳转

语法：`GOTO <数字>;` → 跳转到对应的 `N<数字>;` 标签行。

```macro
GOTO 100;     (* Ctrl+Click "100" → 跳转到 N100; *)
...
N100;         (* 目标标签 *)
G00 X0;
```

**说明**：
- 使用 `Ctrl+Click` 或 `F12` 跳转
- `GOTO #变量`（运行期跳转）无法静态验证目标
- 若目标 N 标签不存在，诊断会显示警告

## 6.2 G65 宏程序跳转

语法：`G65 P<程序号>;` → 跳转到对应宏程序文件。

```macro
G65 P1000;    (* Ctrl+Click "P1000" → 打开 G1000.macro *)
```

**文件搜索规则**（按优先级）：

1. 工作区根目录：`G1000`、`G1000.macro`、`G1000.G`、`G1000.scp`
2. 工作区子目录（递归搜索，最大深度 5 层）
3. `syntecMacro.includePath` 配置中的额外路径

**文件名转换规则**：
- 纯数字 `1000` → `G1000`
- 带 G 前缀 `G1000` → `G1000`
- 不足 4 位自动补零：`P100` → `G0100`

# 第七部分 实时语法诊断

## 7.1 诊断机制

- 文件打开时自动扫描
- 文件修改后 300ms 防抖重新扫描
- 支持通过 `syntecMacro.enableDiagnostics` 开关

## 7.2 诊断规则一览

### 错误（Error）级别

| 检查项 | 触发条件 | 错误消息 |
|--------|----------|----------|
| **中文字符** | 代码中出现中文字符（注释和字符串内除外） | `中文字符：宏程序只允许使用英文字符` |
| **中文标点** | 出现中文标点 `；：，。！？【】《》（）""''、` | `中文标点 "X"：宏程序应使用英文字符` |
| **IF 不匹配** | `END_IF` 没有对应的 `IF` | `END_IF 没有匹配的 IF` |
| **FOR 不匹配** | `END_FOR` 没有对应的 `FOR` | `END_FOR 没有匹配的 FOR` |
| **WHILE 不匹配** | `END_WHILE` 没有对应的 `WHILE` | `END_WHILE 没有匹配的 WHILE` |
| **CASE 不匹配** | `END_CASE` 没有对应的 `CASE` | `END_CASE 没有匹配的 CASE` |
| **REPEAT 不匹配** | `UNTIL` 没有对应的 `REPEAT` | `UNTIL 没有匹配的 REPEAT` |
| **ELSE 不匹配** | `ELSE` 没有对应的 `IF` 或 `CASE` | `ELSE 没有匹配的 IF 或 CASE` |
| **ELSEIF 不匹配** | `ELSEIF` 没有对应的 `IF` | `ELSEIF 没有匹配的 IF` |
| **ELSE 后接 ELSEIF** | `IF` 块已有 `ELSE`，又出现 `ELSEIF` | `IF 块已有 ELSE，再次遇到 ELSEIF` |
| **ELSIF 不支持** | 使用 `ELSIF` 关键字 | `ELSIF 不支持，请使用 ELSEIF` |

### 警告（Warning）级别

| 检查项 | 触发条件 | 警告消息 |
|--------|----------|----------|
| **括号不匹配（多余右括号）** | `)` 没有对应的 `(` | `括号不匹配：多余的右括号` |
| **括号不匹配（缺少右括号）** | `(` 没有对应的 `)` | `括号不匹配：缺少 N 个右括号` |
| **块未关闭** | 文件结束时 `IF`/`FOR`/`WHILE`/`CASE`/`REPEAT` 块未关闭 | `XXX 块缺少对应的 END_（文件结束）` |
| **GOTO 目标不存在** | `GOTO 999;` 但文件中无 `N999;` | `GOTO 目标 999 不存在` |
| **缺少 %@MACRO** | 文件首行以 `%` 开头但无 `%@MACRO` | `此文件缺少 %@MACRO 文件头，将被视为 ISO 格式文件` |

## 7.3 诊断处理流程

```
文件内容 → 逐行扫描
  ├── 第一遍：收集 N 标签 + 检查 %@MACRO 头
  └── 第二遍：逐行处理
       ├── 中文字符/标点检查
       ├── 括号匹配检查
       ├── 关键字位置提取
       │    ├── 长关键字替换占位（防 ENDREPEAT 内 REPEAT 误匹配）
       │    └── 短关键字匹配
       └── 按位置排序处理关键字
            ├── 开启关键字 → 入栈
            ├── 关闭关键字 → 出栈匹配
            ├── ELSE/ELSEIF → 验证配对
            ├── UNTIL → 关闭 REPEAT
            └── EXIT → 标记循环已退出
```

## 7.4 诊断示例

```macro
(* 示例 1：括号不匹配 *)
#1 := ABS(#2 + ;          (* 警告：缺少 1 个右括号 *)

(* 示例 2：块不匹配 *)
IF #1 > 0 THEN
  #2 := 1;
                            (* 警告：IF 块缺少对应的 END_ *)

(* 示例 3：中文字符 *)
#1 := 变量;                (* 错误：中文字符 *)
#2 := 100；                (* 错误：中文标点 "；" *)

(* 示例 4：ELSIF 不支持 *)
ELSIF #1 > 0 THEN         (* 错误：ELSIF 不支持，请使用 ELSEIF *)

(* 示例 5：GOTO 目标不存在 *)
GOTO 999;                   (* 警告：GOTO 目标 999 不存在 *)
```

# 第八部分 代码片段

输入前缀 + `Tab` 快速生成代码块。

## 8.1 控制流片段

| 前缀 | 名称 | 生成内容 |
|------|------|----------|
| `if` | IF 语句（简化版） | `IF 条件 THEN ... END_IF` |
| `ife` | IF 语句（完整版） | `IF ... ELSEIF ... ELSE ... END_IF` |
| `for` | FOR 循环 | `FOR #i = 1 TO 10 BY 1 DO ... END_FOR` |
| `while` | WHILE 循环 | `WHILE 条件 DO ... END_WHILE` |
| `repeat` | REPEAT 循环 | `REPEAT ... UNTIL 条件 END_REPEAT` |
| `case` | CASE 选择 | `CASE #变量 OF ... DEFAULT: ... END_CASE` |
| `goto` | GOTO 跳转 | `GOTO 标签` |

## 8.2 变量与赋值片段

| 前缀 | 名称 | 生成内容 |
|------|------|----------|
| `set` | 变量赋值 | `#变量 := 值` |
| `setc` | 带注释赋值 | `#变量 := 值  // 注释说明` |
| `bc` | 块注释 | `(* 注释内容 *)` |

## 8.3 程序模板片段

| 前缀 | 名称 | 说明 |
|------|------|------|
| `macro` | 完整 MACRO 程序模板 | 含 `%@MACRO` 头、文件信息注释、变量声明区、主程序、M99 结尾 |
| `sub` | 子程序模板 | 含 `%@MACRO` 头、参数说明、子程序代码、M99 返回 |

## 8.4 函数片段

| 前缀 | 名称 | 生成内容 |
|------|------|----------|
| `alarm` | ALARM 报警 | `ALARM(报警编号, "报警信息")` |
| `msg` | MSG 消息提示 | `MSG(" 提示信息")` |
| `wait` | WAIT 等待 | `WAIT(等待时间毫秒)` |
| `sysvar` | SYSVAR 系统变量 | `#变量 := SYSVAR("系统变量名")` |
| `param` | PARAM 参数读取 | `#变量 := PARAM(参数类型, 参数编号)` |
| `chkmn` | CHKMN 检查 M 码 | `#变量 := CHKMN(M码编号)` |
| `push` | PUSH 压栈 | `PUSH(值)` |
| `pop` | POP 出栈 | `#变量 := POP()` |

## 8.5 I/O 操作片段

| 前缀 | 名称 | 生成内容 |

# 第九部分 控制器语法规则

以下规则已通过新代控制器实测确认，扩展严格按照这些规则进行语法检查和补全。

## 9.1 基本语法规则

| 规则 | 说明 | 正确示例 | 错误示例 |
|------|------|----------|----------|
| **文件头** | 首行必须为 `%@MACRO` | `%@MACRO` | `%` 或无 |
| **行尾分号** | 除特定语法外，每行必须以 `;` 结尾 | `#1 := 100;` | `#1 := 100` |
| **赋值运算符** | 使用 `:=` | `#1 := 100;` | `#1 = 100;` |
| **注解** | 行注解 `//`，块注解 `(* ... *)` | `// 注释` / `(* 注释 *)` | `( 注释 )` |

## 9.2 控制流语法规则

| 规则 | 说明 |
|------|------|
| **GOTO 语法** | `GOTO 100;`（不带 N），目标为 `N100;` |

| **ELSIF 不支持** | 请使用 `ELSEIF` |
| **INT() 不支持** | 请使用 `FLOOR()` 或 `CEIL()` |
| **IF 条件** | 支持 `IF #1 = 1 THEN`（可无括号）和 `IF (#1 = 1) THEN`（有括号） |
| **CASE 冒号语法** | `<值>: <语句>` 冒号分隔 |

# 第十部分 故障排除

## 问题 1：语法高亮不工作

**原因**：文件类型未正确识别。

**解决方法**：
1. 确认文件扩展名为 `.macro`、`.G`、`.scp` 或 `.nc`
2. 或确认文件首行为 `%@MACRO`
3. 手动选择语言：右下角点击语言模式 → 选择 `Syntec 新代宏程序`

## 问题 2：补全不弹出

**原因**：补全被禁用或触发条件未满足。

**解决方法**：
1. 检查配置：`syntecMacro.enableCompletions` 是否为 `true`
2. 手动触发：`Ctrl+Space`
3. 确认光标前有字母（如输入 `ST` 再触发）

## 问题 3：G65 跳转不工作

**原因**：目标文件不在搜索路径中。

**解决方法**：
1. 确认目标文件存在于工作区中
2. 文件名约定：`G1000.macro`（程序号 1000 → 文件名 G1000）
3. 配置 `syntecMacro.includePath` 添加额外搜索路径
4. 检查诊断：目标不存在时会显示警告

## 问题 4：诊断报错但代码正确

**原因**：可能遇到了边缘情况。

**解决方法**：
1. 检查是否是中文标点符号（如 `；` 应为 `;`）
2. 检查是否是控制器不支持的语法（如 `ELSIF`）
3. 提交 Issue 到 GitHub，附上代码示例

## 问题 5：`#` 变量前显示黑色/彩色方块

**原因**：VSCode 将 `#100` 等变量误识别为 CSS 颜色值。

**解决方法**：
1. 扩展已在 v2.4.0 中自动禁用颜色装饰器
2. 如仍有问题，手动在设置中搜索 `colorDecorators`，设为 `false`
3. 重新加载窗口（`Ctrl+Shift+P` → `Reload Window`）

## 问题 6：扩展无法激活

**原因**：VSCode 版本过低。

**解决方法**：
- 扩展要求 VSCode >= 1.60.0
- 更新 VSCode 到最新版本

| `END_IF` | 条件判断结束 | — |

## D.2 循环控制

| 关键字 | 说明 | 配对结束 |
|--------|------|----------|
| `FOR` | 计数循环开始 | `END_FOR` |
| `TO` | FOR 循环终止值 | — |
| `BY` | FOR 循环步进值 | — |
| `DO` | FOR/WHILE 循环体开始 | — |
| `END_FOR` | FOR 循环结束 | — |
| `WHILE` | 条件循环开始 | `END_WHILE` |
| `END_WHILE` | WHILE 循环结束 | — |
| `REPEAT` | 重复循环开始 | `END_REPEAT` |
| `UNTIL` | REPEAT 循环条件 | — |
| `END_REPEAT` | REPEAT 循环结束 | — |
| `EXIT` | 跳出当前循环 | — |

## D.3 CASE 选择

| 关键字 | 说明 |
|--------|------|

# 附录 F 运算符优先级

| 优先级 | 运算符 | 说明 |
|--------|--------|------|
| 1 | `( )` `[ ]` | 括号 |

| 4 | `*` `/` `MOD` | 乘、除、模数 |
| 5 | `+` `-` | 加、减 |

| 7 | `=` `<>` | 等于、不等于 |
| 8 | `&` `AND` | 布尔"且" |
| 9 | `XOR` | 布尔"互斥" |
| 10 | `OR` | 布尔"或" |
