## 2. 已核实的关键边界

### `ALARM` 与 `MSG`

- 数值 ID 均只能是 `0~65535` 的整数；越界或非整数会触发 `COR-023`。
- `ALARM` 需要系统复位才可清除；`MSG` 可用 `ESC` 清除，并会在程序结束时自动消失。
- `MSG("text")` 的预设 ID 与控制器版本有关：`10.118.48U`、`10.118.52O`、`10.118.56I`、`10.118.60C`、`10.118.62` 及之后为 `65535`，之前版本为 `-1`。
- 提示文字长度有限制：中文 19 个字、英文 39 个字。由于字符编码、混合文本与版本因素，暂不做静态长度 error。

### `WAIT()` 与 `SLEEP()`

- `WAIT()` 解决 MACRO 预解导致的“读取早于运动完成”问题；它不是一般性的子程序等待机制。
- `M98`、`M99`、`M198` 不在 `WAIT()` 的等待保证范围内。
- `SLEEP()` 用于循环内让出执行权，避免核心持续解译造成资源占用或人机响应问题；它不替代 `WAIT()` 的运动同步语义。

### `CHK*`

- `CHKMN`、`CHKSN`、`CHKMT`、`CHKMI` 以字符串代码进行比对，结果为 `1`（一致）或 `0`（不符）。
- `CHKINF` 用类别编号和字符串代码比对；类别范围 `1~5` 已被现有静态诊断覆盖。
- 这些函数的真值取决于目标控制器信息，插件只应校验可确定的参数形态，不应尝试判断运行时比对结果。
