## 2. 已核实的关键边界

### `STD` 与 `STDAX`

- `STD` 用 `Pr17` 的控制精度把整数数值标准化为输入单位；`#1600` 是线性轴最小输入单位（LIU），通常作为第二引数。
- 对长度或角度的整数引数，官方 MACRO 手册建议在运算前使用 `STD` 标准化，避免控制精度导致的单位歧义。
- `STDAX` 的第二引数为轴向名称。小数点型态输入不会发生数值或型别转换；若输入为整数且 `Pr3241=1`，函数不进行数值转换，只转为小数点型态。
- `Pr17`、`Pr3241`、单位制和可用轴均属控制器配置，静态分析不能可靠给出结果或错误。

### `SQRT`

- `SQRT` 的引数须大于或等于 `0`；负数会触发 `RS-008` 值域错误。
- 插件仅对负数常量报告 `FUNCTION_MATH_DOMAIN`，不推断变量、表达式或跨行赋值的实际值。

### `ACOS` 与 `ASIN`

- `ACOS` 与 `ASIN` 的引数须位于 `-1~1`；常量越界会触发 `RS-008` 值域错误。
- 插件只对可判定的数值常量报告 `FUNCTION_MATH_DOMAIN`，包含 `-1` 与 `1` 的边界值不会报错。

### `MAX`、`MIN`、`SIGN` 与 `RANDOM`

- `MAX` 和 `MIN` 分别从两个输入值中选择最大值和最小值；目前没有来自函数表的固定定义域或静态失败条件。
- `SIGN` 依据输入的正负号回传 `-1`、`0` 或 `1`；`RANDOM()` 产生 `0~32767` 间的随机数。
- 这些结果取决于运行时输入或随机状态，插件仅提供补全和 hover，不推断返回值或新增诊断。

### `CEIL`、`FLOOR` 与 `ROUND`

- `CEIL` 回传大于或等于输入值的最小整数；`FLOOR` 回传小于或等于输入值的最大整数。
- `ROUND` 回传四舍五入后的值，例如 `ROUND(1.4)` 为 `1`、`ROUND(1.5)` 为 `2`。
- 函数表未给出独立的静态失败条件，因此插件仅提供补全和 hover，不新增诊断。

### 基础数学函数

- `ABS` 回传绝对值；`SIN`、`COS` 和 `TAN` 使用角度制，函数表给出 `SIN(90)=1`、`COS(180)=-1` 和 `TAN(45)=1`。
- `ATAN` 回传 ±90° 范围的角度制反正切；`EXP` 以自然数 `e` 为底计算指数。
- 当前资料没有提供可跨版本静态化的错误条件或完整版本基线，插件仅固定 hover 语义，不新增诊断。

### `LN` 与 `POW`

- `LN` 的引数须为正数；非正数会触发 `RS-008` 值域错误。
- `POW` 的基底不可为负数；MACRO 函数表将该错误列为 `COR-122`。
- 插件仅对可判定的数值常量报告 `FUNCTION_MATH_DOMAIN`，不推断变量、表达式或跨行赋值的实际值。

### 栈函数

- `PUSH` 依压入顺序建立栈；最后压入的值位于栈顶。
- `POP` 从栈顶读取并移除资料。
- `STKTOP[index]` 是非破坏性读取；索引以 `0` 为栈顶，例如依序 `PUSH(5)`、`PUSH(6)`、`PUSH(7)` 后，`STKTOP[0]`、`STKTOP[1]`、`STKTOP[2]` 分别为 `7`、`6`、`5`。
- 函数表未在本轮检索结果中提供跨版本稳定的栈深度、空栈或越界失败行为，故不新增诊断。
